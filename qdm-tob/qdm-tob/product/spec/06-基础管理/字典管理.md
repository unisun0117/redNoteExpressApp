# 系统字典管理

> V1.0 ｜ 2026-06-25 ｜ 待评审

---

## 1. 背景与目标

### 一句话背景

系统各功能模块中存在大量可自定义的枚举值（如：客户等级、订单类型、支付方式等），需要统一的管理入口，避免枚举值散落在代码或配置文件中，让运营和产品人员可自助维护。

### 核心目标

| 维度 | 目标 |
|------|------|
| 统一管理 | 所有系统的枚举值集中维护，一处定义、全局生效 |
| 自助配置 | 运营人员无需开发介入即可新增、修改字典值 |
| 开发提效 | 前端/后端通过字典编码直接拉取枚举列表，减少硬编码 |

---

## 2. 角色与使用场景

### 目标用户

| 角色 | 说明 |
|------|------|
| 系统管理员 | 负责字典的创建和维护 |
| 运营人员 | 日常维护字典项（如新增一个客户等级选项） |
| 开发人员 | 接入字典接口，在功能模块中消费字典数据 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 接入新模块 | 开发新功能（如"客户标签"）时，在字典管理里新建一个字典编码，添加对应的字典项，前端通过接口拉取下拉选项 |
| 2 | 业务调整 | 需要给"订单类型"增加一个选项，进入字典管理 → 搜索对应字典 → 新增一行字典项 → 保存即生效 |
| 3 | 禁用旧选项 | 某个字典项不再使用但历史数据需保留，将其状态改为"停用"，前端下拉不再展示但历史记录仍可关联 |
| 4 | 批量导出 | 需要导出全部字典配置做线下审核，使用导出功能按当前筛选条件导出 Excel |

## 3. 核心业务流程

### 主干流程

```
进入字典管理页面
  → 搜索目标字典（按字典名称模糊搜索）
    → 查看该字典下的所有字典项
      → 可执行：新增字典项 / 编辑字典项 / 停用字典项
```

### 状态流转

字典项仅涉及简单的启用/停用，无复杂状态机：

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| 启用 | 正常使用，前端可拉取 | 新增时默认；或从停用手动恢复 |
| 停用 | 隐藏不可选，历史数据保留 | 手动停用 |

### 异常流

| 场景 | 处理 |
|------|------|
| 新增时字典编码+字典项编码重复 | 后端校验唯一性，前端提示"该字典项编码已存在" |
| 字典编码/字典项编码包含特殊字符 | 前端限制仅允许字母、数字、下划线 |
| 停用已被引用的字典项 | 提示"该字典项被 X 个功能引用，停用后相关页面将不再展示该选项"，确认后执行 |

## 4. 界面与交互说明

### 界面布局

```
┌──────────────────────────────────────────────┐
│  搜索区                                       │
│  [字典编码] [字典名称]                             │
│  [查询] [重置]                  [新增] [导出]  │
├──────────────────────────────────────────────┤
│  列表区（分页）                                 │
│  字典编码 | 字典名称 | 用途描述 | 明细项数 | 操作      │
│  ──────────────────────────────────────────── │
│  ...                                          │
│  [分页器]                                      │
└──────────────────────────────────────────────┘
```

### 交互动作

| 动作 | 触发方式 | 响应 |
|------|----------|------|
| 查询 | 点击"查询"按钮或回车 | 列表按条件刷新 |
| 重置 | 点击"重置"按钮 | 清空搜索条件，恢复默认列表 |
| 新增 | 点击"新增"按钮 | 弹出新增弹窗（见 4.2） |
| 编辑 | 点击行操作列的"编辑" | 弹出编辑弹窗（见 4.3） |
| 导出 | 点击"导出"按钮 | 按当前查询条件导出 Excel |

### 新增弹窗

```
┌─────────────────────────────────────┐
│  新增字典项                          │
│                                     │
│  字典编码 *   [________]             │
│  字典名称 *   [________]             │
│  用途描述     [________]             │
│                                     │
│  ┌─ 字典项明细 ──────────────────┐  │
│  │ 字典项编码  字典项名称  顺序  状态│  │
│  │ [____]     [____]    [__]  启用 │  │
│  │ [____]     [____]    [__]  启用 │  │
│  │ [+ 添加一行]                    │  │
│  └────────────────────────────────┘  │
│                                     │
│          [取消]  [保存]              │
└─────────────────────────────────────┘
```

**规则**：
- 字典编码和字典名称必填
- 字典项明细至少添加一行
- 字典项编码在同一字典编码下不可重复
- 保存时同时写入字典主表和字典项明细

### 编辑弹窗

```
┌─────────────────────────────────────┐
│  编辑字典项                          │
│                                     │
│  字典编码      [ORDER_TYPE] (不可改)  │
│  字典名称 *    [订单类型]             │
│  用途描述      [用于订单下单时的类型选择]│
│                                     │
│  ┌─ 字典项明细 ──────────────────┐  │
│  │ 字典项编码  字典项名称  顺序  状态│  │
│  │ NORMAL(灰)  普通订单    [1]  启用 │  │
│  │ URGENT(灰)  加急订单    [2]  启用 │  │
│  │ PRE(灰)     预售订单    [3]  停用 │  │
│  └────────────────────────────────┘  │
│                                     │
│          [取消]  [保存]              │
└─────────────────────────────────────┘
```

**规则**：
- 字典编码不可修改（灰显）
- 已有字典项编码不可修改（灰显），新增的字典项编码可写
- 可修改：字典名称、用途描述、字典项名称、顺序、状态
- 编辑时支持新增字典项，不提供删除已有字典项（已停用即可），新增未保存的字典项可删除

### 极限状态

| 状态 | 处理 |
|------|------|
| 空数据 | 列表区展示"暂无数据"占位图，引导点击"新增"创建第一条 |
| 加载中 | 列表区展示骨架屏或加载动画 |
| 数据极多 | 分页展示，默认每页 20 条，支持切换 10/20/50/100 |
| 输入校验失败 | 字段下方红色提示文案，弹窗不关闭 |

## 5. 数据字典与字段级规则

### 列表字段

| 字段名称 | 类型 | 来源 | 默认值 | 读写 | 校验规则 | 说明 |
|----------|------|------|--------|------|----------|------|
| 字典编码 | String(50) | 用户输入 | — | 新增可写/编辑只读 | 必填，仅允许字母数字下划线，不可与已有字典编码重复 | 逻辑主键 |
| 字典名称 | String(100) | 用户输入 | — | 可编辑 | 必填，最长 100 字符 | — |
| 用途描述 | String(200) | 用户输入 | — | 可编辑 | 最长 200 字符 | — |

> **说明**：列表仅展示字典的主信息。字典的状态由字典项明细的状态决定（全部停用即视为字典停用）。字典项明细（字典项编码、字典项名称、顺序、状态）在新增/编辑弹窗中维护。

### 搜索字段

| 搜索项 | 匹配方式 | 说明 |
|--------|----------|------|
| 字典编码 | 模糊匹配 | LIKE %keyword% |
| 字典名称 | 模糊匹配 | LIKE %keyword% |

> **说明**：列表展示字典的主信息。点击"编辑"进入弹窗后，可查看和维护该字典下的所有字典项明细（字典项编码、字典项名称、顺序、状态）。

### 编辑权限矩阵

| 字段 | 新增时 | 编辑时 |
|------|--------|--------|
| 字典编码 | ✅ 可写 | ❌ 只读 |
| 字典名称 | ✅ 可写 | ✅ 可写 |
| 用途描述 | ✅ 可写 | ✅ 可写 |
| 字典项编码 | ✅ 可写 | ❌ 只读 |
| 字典项名称 | ✅ 可写 | ✅ 可写 |
| 顺序 | ✅ 可写 | ✅ 可写 |
| 字典项状态 | ✅ 可写（默认启用） | ✅ 可写 |

### 导出规则

- 导出格式：Excel (.xlsx)
- 导出范围：当前查询条件下的所有数据（不受分页限制）
- 导出字段：同列表字段，含字典编码、字典名称、用途描述

## 6. 系统交互与边界

### 前置依赖

无。字典管理是基础数据模块，不依赖其他业务模块。

### 上下游影响

| 方向 | 模块 | 影响 |
|------|------|------|
| 提供 → | 所有业务模块 | 各模块通过字典编码拉取字典项列表，用于下拉框、单选、多选等场景 |
| 提供 → | 前端全局 | 前端启动时可预加载高频字典到本地缓存，减少重复请求 |
| 依赖 ← | 无 | — |

### 接口概要

| 接口 | 方法 | 说明 |
|------|------|------|
| `/dict/list` | GET | 分页查询字典项列表，支持模糊搜索 |
| `/dict/{code}/items` | GET | 按字典编码获取所有启用字典项（供前端下拉使用） |
| `/dict/create` | POST | 新增字典（含字典项） |
| `/dict/update` | PUT | 编辑字典及字典项 |
| `/dict/export` | GET | 按查询条件导出 Excel |

## 7. 非功能性需求

### 性能

| 维度 | 要求 |
|------|------|
| 列表查询 | P95 < 300ms |
| 按编码获取字典项 | P95 < 50ms（需缓存） |
| 导出 | 5,000 条以内 < 10s |

### 权限与安全

| 维度 | 要求 |
|------|------|
| 数据权限 | 无限制，所有有权限的用户看到同一套字典 |
| 操作权限 | 列表查看、导出为通用权限；新增、编辑需管理员角色 |
| 缓存策略 | 字典项接口结果缓存 10 分钟，新增/编辑后主动刷新缓存 |
