> **更新日期**：2026-07-03

## 1. 背景与目标 (Background & Objectives)

**背景**：平台后台管理系统和企业微信 H5 端各自拥有独立的菜单体系和操作按钮，需要统一的菜单管理入口对两端菜单进行集中维护。菜单是 RBAC 权限模型的基础数据，角色通过绑定菜单获得对应的页面访问和按钮操作权限。

**目标**：为系统管理员提供菜单的全生命周期管理能力，支持后台和企微两端的菜单树维护，按"菜单 → 页面 → 按钮"三级结构组织，按钮可与权限组关联，为后续角色权限配置提供数据基础。

**一句话背景**：统一管理后台和企微两端的菜单结构，为角色权限配置提供可绑定的菜单数据。

## 2. 角色与使用场景 (Roles & Scenarios)

### 目标用户

| 角色 | 说明 |
| --- | --- |
| 系统管理员 | 负责菜单的新增、编辑、删除，以及按钮与权限组的绑定 |
| 普通用户 | 通过被分配的角色，间接使用菜单管理配置的页面和按钮 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 新建菜单 | 在后台或企微菜单树下新增一个一级菜单目录，填写菜单编码、名称和排序 |
| 2 | 新建页面 | 在某个菜单下新增页面，后续可在此页面下添加操作按钮 |
| 3 | 新建按钮并绑定权限 | 在某个页面下新增操作按钮，并为其绑定权限组，用于后续角色权限配置 |
| 4 | 编辑已有节点 | 编辑某个菜单/页面的名称或顺序，或编辑按钮的名称和权限组绑定，编码不可修改 |
| 5 | 查看菜单结构 | 通过左侧树形结构浏览后台或企微的完整菜单体系 |
| 6 | 删除按钮 | 删除某个页面下的按钮，删除前需二次确认 |

## 3. 核心业务流程 (Core Business Flow)

### 3.1 整体层级关系

```
菜单（一级目录）
  └── 页面（二级页面）
        └── 按钮（三级操作按钮，绑定权限组）
```

### 3.2 新增菜单流程

1. 在左侧菜单树顶部切换"后台菜单"或"企微菜单"
2. 在树结构的根节点或目录节点上点击"新增菜单"
3. 右侧弹出菜单编辑表单，填写菜单编码（唯一）、菜单名称（唯一）、顺序（整数）
4. 点击"保存"，系统校验唯一性和必填字段
5. 校验通过后，新菜单出现在左侧树结构中

### 3.3 新增页面流程

1. 在左侧菜单树中选中目标菜单节点
2. 点击"新增页面"
3. 右侧弹出页面编辑表单，填写页面编码（唯一）、页面名称（唯一）、顺序（整数）
4. 点击"保存"，新页面出现在选中菜单节点的下方

### 3.4 新增按钮流程

1. 在左侧菜单树中点击目标页面节点
2. 右侧展示该页面的详情和已有的按钮列表
3. 点击"新增按钮"
4. 填写按钮编码（唯一）、按钮名称（唯一），并从权限组列表中多选绑定
5. 点击"保存"，新按钮出现在该页面的按钮列表中

### 3.5 编辑流程

1. 在左侧菜单树中点击目标节点（菜单/页面/按钮）
2. 右侧回填该节点的当前信息
3. **编码不可修改**（灰显只读），其他字段可编辑
4. 点击"保存"，更新节点信息

### 3.6 删除按钮流程

1. 在左侧菜单树中点击目标页面节点，右侧展示该页面的按钮列表
2. 点击目标按钮行末的"删除"操作
3. 系统弹出二次确认弹窗，提示"确认删除该按钮？"
4. 用户确认后，系统删除该按钮并刷新右侧按钮列表
5. 用户取消则关闭弹窗不做任何操作

### 3.7 状态流转

菜单/页面/按钮的状态均为简单的启用/禁用，不涉及复杂状态机：

| 状态 | 含义 | 触发条件 |
| --- | --- | --- |
| 启用 | 正常使用，角色可绑定 | 新增时默认 |
| 禁用 | 隐藏不可选，已绑定角色的菜单不展示 | 手动禁用 |

> **说明**：禁用父节点后，其下所有子节点（页面、按钮）在角色菜单配置中也不可勾选。

### 4.9 极限状态

| 状态 | 处理 |
| --- | --- |
| 菜单树为空 | 左侧树区展示空状态，引导点击"新增菜单"创建第一条 |
| 页面下无按钮 | 右侧按钮列表展示"暂无按钮"，引导点击"新增按钮" |
| 权限组列表为空 | 权限组多选框区域提示"暂无可用权限组，请先维护权限定义" |
| 数据极多 | 菜单树一次性加载全部展开，无分页（单端菜单总量通常 < 500 节点） |
| 输入校验失败 | 字段下方红色提示文案，保存按钮保持可用但拦截提交 |
| 编码重复 | 提示"该菜单编码/页面编码/按钮编码已存在" |
| 名称重复 | 提示"该菜单名称/页面名称/按钮名称已存在" |
| 删除确认 | 点击删除按钮后弹出确认弹窗，确认后执行、取消则关闭 |

## 5. 数据字典与字段级规则 (Data & Field Rules)

### 5.1 菜单核心字段

| 字段名称 | 字段类型 | 来源/依赖 | 默认值 | 读写权限 | 校验规则与约束 | 说明/占位符 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 菜单编码 | String(60) | 新增时输入 | — | 新增可写/编辑只读 | 必填，菜单级别唯一，仅允许字母数字下划线 | 如 WORKSPACE、PRODUCT |
| 菜单名称 | String(60) | 新增时输入 | — | 可编辑 | 必填，菜单级别唯一，最长 60 字符 | 如"工作台"、"商品管理" |
| 顺序 | Integer | 新增时输入 | 0 | 可编辑 | 必填，整数，越小越靠前 | 控制同级节点排序 |
| 状态 | Enum | 配置 | 启用 | 可编辑 | 枚举：启用/禁用 | 禁用后子节点全部不可用 |
| 所属分组 | Enum | 顶部切换 | 后台菜单 | 只读 | 后台菜单 / 企微菜单 | 由菜单类型切换决定 |
| 创建时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动生成 |
| 更新时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动更新 |

### 5.2 页面核心字段

| 字段名称 | 字段类型 | 来源/依赖 | 默认值 | 读写权限 | 校验规则与约束 | 说明/占位符 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 页面编码 | String(60) | 新增时输入 | — | 新增可写/编辑只读 | 必填，页面级别唯一，仅允许字母数字下划线 | 如 HOME、WAREHOUSE_LIST |
| 页面名称 | String(60) | 新增时输入 | — | 可编辑 | 必填，页面级别唯一，最长 60 字符 | 如"首页"、"商品资料" |
| 顺序 | Integer | 新增时输入 | 0 | 可编辑 | 必填，整数 | 控制同级页面排序 |
| 状态 | Enum | 配置 | 启用 | 可编辑 | 枚举：启用/禁用 | — |
| 父菜单编码 | String | 树节点位置 | — | 只读 | 由用户点击的父菜单决定 | 展示所属菜单名称 |
| 创建时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动生成 |
| 更新时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动更新 |

### 5.3 按钮核心字段

| 字段名称 | 字段类型 | 来源/依赖 | 默认值 | 读写权限 | 校验规则与约束 | 说明/占位符 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 按钮编码 | String(60) | 新增时输入 | — | 新增可写/编辑只读 | 必填，当前页面下唯一，仅允许字母数字下划线 | 如 VIEW、CREATE、EDIT |
| 按钮名称 | String(60) | 新增时输入 | — | 可编辑 | 必填，当前页面下唯一，最长 60 字符 | 如"查看"、"新增"、"编辑" |
| 顺序 | Integer | — | — | — | — | 按钮不设顺序，按创建先后排列 |
| 状态 | Enum | 配置 | 启用 | 可编辑 | 枚举：启用/禁用 | — |
| 权限组 | Array | 权限定义模块 | — | 可编辑 | 必填，至少选一个 | 多选，用于角色绑定时的权限匹配 |
| 父页面编码 | String | 树节点位置 | — | 只读 | 由用户点击的父页面决定 | 展示所属页面名称 |
| 创建时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动生成 |
| 更新时间 | DateTime | 系统记录 | 当前时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自动更新 |

### 5.4 展示逻辑

- 菜单节点：📁 图标 + 菜单名称
- 页面节点：📄 图标 + 页面名称
- 按钮节点：🔘 图标 + 按钮名称
- 树结构默认全部展开
- 菜单类型切换（后台/企微）时，树结构整体刷新
- 点击树中不同节点时，右侧区域平滑切换内容

### 5.5 保存逻辑

- 新增：校验唯一性 → 创建记录 → 刷新左侧树
- 编辑：校验唯一性（排除自身）→ 更新记录 → 刷新左侧树中对应节点名称
- 删除按钮：二次确认 → 删除按钮记录 → 刷新右侧按钮列表
- 菜单/页面被禁用时，其下所有子节点级联不可用

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 权限与安全

| 维度 | 要求 |
| :--- | :--- |
| 数据权限 | 无限制，所有有权限访问菜单管理的用户看到同一套菜单树 |
| 操作权限 | 查看为通用权限；新增、编辑需具备菜单管理操作权限 |
| 编码安全 | 菜单/页面/按钮编码一旦创建不可修改，防止权限体系关联断裂 |

### 7.2 性能要求

| 维度 | 要求 |
| :--- | :--- |
| 菜单树加载 | 一次性加载完整树结构，响应流畅不卡顿 |
| 节点切换 | 左侧点击节点后右侧内容即时展示，无明显延迟 |
| 保存操作 | 保存完成后及时反馈结果并刷新树 |

### 7.3 业务规则

- 菜单编码和菜单名称在菜单级别唯一，页面编码和页面名称在页面级别唯一，按钮编码和按钮名称在所属页面下唯一
- 编码创建后不可修改，名称可修改但需保证唯一
- 菜单树为三级结构：菜单（一级）→ 页面（二级）→ 按钮（三级），不允许跨级新增
- 禁用父节点后子节点级联不可用
- 菜单和页面不支持删除，仅按钮支持单条删除，需二次确认
- 菜单类型分为后台菜单和企微菜单，两者数据隔离，互不影响
- 按钮可绑定多个权限组，保存时全量覆盖
- 同级节点的排列顺序由"顺序"字段从小到大决定

### 8.3 特殊业务场景

**场景一：后台与企微菜单隔离**

后台菜单和企微菜单展示上各自独立，切换菜单类型时树结构整体刷新，右侧编辑区清空。但菜单编码、页面编码、按钮编码在全局范围内（跨后台和企微）保持唯一，不允许同一编码在两端重复使用。

**场景二：级联禁用**

禁用一个菜单节点时，其下所有页面和按钮在角色菜单配置中均不可勾选。重新启用后子节点跟随恢复。

**场景三：权限组解绑**

当权限定义模块中某个权限组被删除时，已绑定该权限组的按钮不受影响，但在编辑时该权限组不再出现在可选项列表中。建议在删除权限组时提示引用数量。
