## 1. 背景与目标 (Background & Objectives)

**背景**：平台面向不同销售大区、不同客户群体存在差异化定价需求。通过设置价格组，将客户归入不同价格组，同一商品在不同价格组下可维护不同售价。客户建档时绑定价格组，小程序端下单时自动匹配对应价格组的售价。

**目标**：为运营管理员提供价格组定义和价格组明细维护能力，支持按销售大区 + 价格组 + 商品条码维度精细化管理商品售价，价格变更超阈值时走审批流程。

**一句话背景**：按销售大区维度定义价格组，按价格组 + 商品条码维护售价，客户下单时按绑定价格组自动匹配价格。

## 2. 角色与使用场景 (Roles & Scenarios)

### 目标用户

| 角色 | 说明 |
| --- | --- |
| 运营管理员 | 负责创建和管理价格组，维护各价格组下商品的售价 |
| 区域运营经理 | 按数据权限管理管辖大区内的价格组和价格明细 |
| 审批人 | 当价格变动比率超过阈值时，审批价格变更申请 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 管理价格组 | 按销售大区创建、编辑价格组（编码+名称+描述） |
| 2 | 维护价格明细 | 在价格组下新增、编辑商品售价 |
| 3 | 查询价格明细 | 通过销售大区、价格组名称、商品条码/名称快速检索价格明细 |
| 4 | 导出价格明细 | 导出价格组明细数据用于线下分析 |
| 5 | 导入价格明细 | 批量导入价格明细数据 |
| 6 | 价格变更审批 | 修改售价超过配置阈值时，变更需经审批后方可生效 |

## 3. 核心业务流程 (Core Business Flow)

### 3.1 整体数据流

```
销售大区管理 ──→ 价格组（按大区） ──→ 价格组明细（按价格组+商品条码）
                                        │
                                        ▼
                              客户建档（绑定价格组）
                                        │
                                        ▼
                              小程序下单（匹配价格组售价）
                                        │
                              价格变更超阈值 ──→ 待审批 ──→ 审批通过 ──→ 售价生效
```

### 3.2 价格组查询流程

```
进入商品价格管理页面
    → 默认展示「价格组」Tab
    → 可选选择销售大区、输入价格组名称（模糊）
    → 点击"查询"
    → 列表刷新展示符合条件的价格组（分页）
```

### 3.3 价格组新增流程

```
点击"新增"按钮
    → 弹出新增弹窗
    → 选择销售大区（必选，下拉）
    → 输入价格组编码（必填）
    → 输入价格组名称（必填）
    → 输入描述说明（可为空）
    → 提交保存，校验「销售大区 + 价格组编码」唯一性
    → 保存成功，刷新列表
```

### 3.4 价格组编辑流程

```
在列表中点击目标行的"编辑"按钮
    → 弹出编辑弹窗，回填当前数据
    → 不可修改字段（灰显只读）：销售大区、价格组编码
    → 可修改字段：价格组名称、描述说明
    → 提交保存，刷新列表
```

### 3.5 价格组明细查询流程

```
切换到「价格组明细」Tab
    → 可选选择销售大区、输入价格组名称（模糊）、输入商品条码/名称（模糊）
    → 点击"查询"
    → 列表刷新展示符合条件的价格明细（分页）
```

### 3.6 价格组明细新增流程

```
点击"新增"按钮
    → 弹出新增弹窗
    → 选择销售大区（必选，下拉）
    → 选择价格组名称（联动销售大区，下拉仅展示该大区下的价格组）
    → 输入商品条码（必填，回车校验在商品资料中存在）
    → 商品名称自动带出（根据商品条码反查）
    → 输入售价（必填，最多两位小数，且大于 0）
    → 提交保存，校验「销售大区 + 价格组编码 + 商品条码」唯一性
    → 保存成功，刷新列表
```

### 3.7 价格组明细编辑流程

```
在列表中点击目标行的"编辑"按钮
    → 弹出编辑弹窗，回填当前数据
    → 不可修改字段（灰显只读）：销售大区、价格组名称
    → 可修改字段：售价
    → 必填字段：变动原因（最多 50 字符）
    → 提交时系统计算价格变动比率
    → 若变动比率 ≤ 配置阈值 → 直接更新售价
    → 若变动比率 > 配置阈值 → 生成待审批记录，审批完成后售价才生效
    → 关闭弹窗，刷新列表
```

### 3.8 价格组明细导出流程

```
点击"导出"按钮
    → 系统按当前查询条件导出 Excel
    → 导出全部列表字段
```

### 3.9 价格组明细导入流程

```
点击"导入"按钮
    → 上传按导出模板格式填写的 Excel
    → 系统按「销售大区 + 价格组编码 + 商品条码」逐行匹配
    → 匹配到已有记录 → 更新售价（变动原因记录为"批量导入"）
    → 未匹配到记录 → 作为新记录新增
    → 校验通过后批量执行，返回导入结果（新增 N 条 / 更新 N 条 / 失败 N 条 / 失败原因）
```

- 变动比率 > 配置阈值 → 生成待审批记录，售价暂不更新，审批完成后才生效

### 4.9 极限状态

| 状态 | 处理 |
| --- | --- |
| 列表为空 | 展示"暂无数据"空状态占位图 |
| 无查询结果 | 展示"未找到匹配的记录" |
| 数据极多 | 列表分页展示，默认每页 20 条 |
| 新增时编码已存在（价格组） | 提示"该销售大区下已存在相同价格组编码" |
| 新增时条码不存在（明细） | 提示"该商品条码在商品资料中不存在，请检查" |
| 新增时明细已存在 | 提示"该价格组下已存在相同商品条码的价格明细" |
| 价格组名称下拉为空 | 提示"该销售大区下暂无价格组，请先创建价格组" |
| 导入失败 | 逐行反馈失败原因 |
| 价格变动超阈值 | 弹窗提示"价格变动比率超过阈值，已生成待审批记录，审批完成后生效" |
| 待审批状态 | 列表中该记录标记为"审批中"，售价保留原值 |

- 「销售大区」下拉选项来源于销售大区管理模块
- 「价格组名称」联动销售大区，下拉仅展示所选大区下的价格组
- 「商品条码」必填，输入后按回车触发校验
  - 校验逻辑：查询商品资料表中是否存在该条码
  - 校验通过 → 自动带出商品名称（只读）
  - 校验失败 → 提示"该商品条码在商品资料中不存在，请检查"
- 「售价」必填，最多两位小数，必须 > 0
- 提交时校验「销售大区 + 价格组编码 + 商品条码」唯一性

### 5.6 价格组明细编辑逻辑

- **不可修改字段**（灰显只读）：销售大区、价格组名称、商品条码、商品名称
- **可修改字段**：售价
- **编辑时必填字段**：变动原因（最多 50 字符）
- 价格变动比率计算：`|新售价 - 原售价| / 原售价 × 100%`
- 变动比率 ≤ 阈值 → 直接更新售价
- 变动比率 > 阈值 → 生成待审批记录（对接审批管理模块），审批完成后售价生效

### 5.7 价格组明细导出逻辑

| 是否导出 | 字段 |
|:---|:---|
| ✅ 导出 | 销售大区、价格组编码、价格组名称、商品条码、商品名称、售价、审批状态、更新人、更新时间 |

### 5.8 价格组明细导入逻辑

- 按模板 Excel 导入，支持新增和更新两种模式
- 匹配键：「销售大区 + 价格组编码 + 商品条码」
- 匹配到已有记录 → 更新售价（变动原因自动记录为"批量导入"）
- 未匹配到记录 → 作为新记录新增
- 导入不触发价格变动审批流程
- 导入完成后展示结果汇总（新增 N / 更新 N / 失败 N / 失败原因）

### 5.9 展示逻辑

- 日期时间格式统一为 `YYYY-MM-DD HH:mm:ss`
- 售价保留 2 位小数展示
- 审批状态列：无需审批（不展示标签）/ 审批中（橙色标签）/ 审批通过（绿色标签）/ 审批驳回（红色标签）
- 价格变动比率结果在编辑保存时弹窗提示

## 6. 系统交互与边界 (System Integrations & Boundaries)

### 6.1 前置依赖

| 依赖系统/模块 | 依赖内容 | 说明 |
| :--- | :--- | :--- |
| 销售大区管理（07-运营管理） | 销售大区下拉数据源 | 价格组和明细页面的销售大区筛选和选择 |
| 商品资料（01-商品管理） | 商品条码校验 + 商品名称反查 | 新增价格明细时校验条码有效性 |
| 审批管理（08-审批管理） | 价格变更审批流程 | 编辑售价超阈值时生成待审批记录 |
| 客户管理（02-客户管理） | 客户价格组绑定 | 客户建档时选择价格组，下单时匹配 |

### 6.2 上下游影响

| 方向 | 模块 | 影响 |
| :--- | :--- | :--- |
| 提供 → | 客户管理（客户建档） | 客户建档时选择价格组的下拉数据源 |
| 提供 → | 小程序下单（价格计算） | 客户下单时按绑定价格组匹配售价 |
| 提供 → | 审批管理 | 价格变更超阈值时推送待审批记录 |
| 依赖 ← | 销售大区管理 | 大区下拉选项数据源 |
| 依赖 ← | 商品资料 | 商品条码反查名称和存在性校验 |

### 6.3 价格匹配数据流

```
客户建档时绑定价格组（客户资料 → 价格组编码）
    → 客户登录小程序浏览商品
    → 系统按「客户绑定的价格组编码 + 商品条码」查询价格组明细
    → 匹配到 → 展示价格组售价
    → 未匹配到 → 需定义兜底策略（后续迭代明确）
```

### 6.4 价格变更审批数据流

```
编辑售价
    → 系统计算变动比率
    → 变动比率 ≤ 阈值 → 直接更新，记录变动原因
    → 变动比率 > 阈值 → 生成待审批记录
        → 审批通过 → 售价生效，更新明细记录
        → 审批驳回 → 售价恢复原值，明细记录不变
```

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 权限与安全

| 维度 | 要求 |
| :--- | :--- |
| 查看权限 | 按用户数据权限过滤销售大区，用户仅能看到管辖大区内的价格组和明细 |
| 编辑权限 | 仅具备运营管理权限的用户可新增、编辑、导入 |
| 导出权限 | 具备查看权限即可导出 |
| 审批权限 | 价格变更审批由审批管理模块统一控制 |

### 7.2 性能要求

| 维度 | 要求 |
| :--- | :--- |
| 列表加载 | 支持分页，默认每页 20 条 |
| 查询响应 | 模糊查询响应时间 < 2s |
| 新增/编辑保存 | 保存操作即时反馈 |
| 导出 | 单次导出上限 10,000 条 |
| 导入 | 单次导入上限 1,000 条 |

### 7.3 业务规则

- 价格组主键为「销售大区 + 价格组编码」，联合唯一
- 价格组明细主键为「销售大区 + 价格组编码 + 商品条码」，联合唯一
- 价格组编码创建后不可修改
- 商品条码必须在商品资料中存在，否则不允许新增明细
- 售价必须大于 0，最多两位小数
- 价格变动比率阈值由系统配置，默认建议 30%
- 编辑售价时必须填写变动原因（最多 50 字符）
- 导入操作不触发价格变动审批流程
- 同一商品可在不同价格组下拥有不同售价
- 价格变更审批期间，原售价保持不变继续使用
