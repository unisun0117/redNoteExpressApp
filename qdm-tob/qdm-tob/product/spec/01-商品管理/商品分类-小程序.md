# 商品分类-小程序 SPEC

> **归属中心**：01-商品管理
> **子模块**：商品分类
> **终端**：小程序端
> **版本**：v1.0
> **更新日期**：2026-07-06
>
> **小程序端**：面向 B 端客户，提供按大分类浏览商品的页面，左侧分类导航 + 右侧商品列表，客户登录并绑定客户后按价格组展示对应售价和库存。

---

## 1. 背景与目标 (Background & Objectives)

**背景**：平台面向不同客户群体存在差异化定价（价格组），客户建档时绑定价格组。小程序分类页作为商品浏览的核心入口，需要根据客户登录状态和绑定的价格组正确展示商品价格和可用库存，未登录或未绑定客户时使用默认大区和价格组兜底。

**目标**：为小程序客户提供按大分类浏览商品的能力，支持搜索商品、查看价格和库存、调整订购数量并加入购物车。

**一句话背景**：左侧大分类导航 + 右侧商品列表，按客户绑定的价格组匹配售价，支持搜索、加购。

## 2. 角色与使用场景 (Roles & Scenarios)

### 目标用户

| 角色 | 说明 |
| --- | --- |
| 访客（未登录/未绑定客户） | 浏览默认大区和默认价格组的商品，搜索商品，加购时引导登录 |
| B 端客户（已登录已绑定） | 浏览所选客户对应价格组的商品，按实际售价下单加购 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 浏览分类商品 | 进入分类页展示左侧大分类列表，点击分类后在右侧展示该分类下的商品列表 |
| 2 | 搜索商品 | 在顶部搜索框输入商品名称关键词搜索商品 |
| 3 | 查看价格和库存 | 展示每个商品的售价（按绑定的价格组匹配）、剩余可用库存 |
| 4 | 加购商品 | 在商品卡片上调整订购数量并加入购物车 |
| 5 | 未登录兜底 | 未登录时展示默认大区和默认价格组的商品和价格 |
| 6 | 切换客户 | 已登录客户切换绑定的客户账号后，商品列表价格和库存按新客户的价格组刷新 |

## 3. 核心业务流程 (Core Business Flow)

### 3.1 页面加载与价格匹配流程

```
打开小程序分类页
    → 判断登录状态
    → 未登录 → 取系统默认配置的大区 + 默认价格组
    → 已登录但未绑定客户 → 取系统默认配置的大区 + 默认价格组
    → 已登录且已绑定客户 → 取客户绑定的价格组 + 所属大区
    → 加载大分类列表（来源于商品分类模块）
    → 默认选中第一个大分类
    → 请求该大分类下的商品列表
        → 查询商品资料（置大区、状态='已上架'）
        → 查询价格组明细（置价格组编码 + 商品条码）获取售价
        → 组装商品卡片数据返回
    → 渲染左侧分类 + 右侧商品列表
```

### 3.2 切换分类流程

```
点击左侧某个大分类
    → 该分类高亮选中
    → 右侧商品列表滚动位置重置到顶部
    → 请求该分类下的商品列表（同上方商品查询逻辑）
    → 右侧商品列表刷新

在右侧商品列表向下滚动到底部
    → 自动切换到下一个大分类（左侧分类导航同步高亮）
    → 已是最后一个分类时不继续切换
    → 右侧商品列表滚动位置重置到顶部
    → 请求下一分类的商品列表并刷新
```

### 3.3 搜索流程

```
在顶部搜索框输入关键词
    → 触发搜索请求（在当前大分类范围内模糊匹配商品名称）
    → 返回匹配的商品列表（含价格和库存）
    → 右侧列表刷新为搜索结果
    → 清空搜索框 → 恢复当前分类下的全部商品
```

### 3.4 加购流程

```
点击商品卡片的加购按钮
    → 未登录 → 引导跳转登录页
    → 已登录 → 弹出数量选择器（默认订购基数，步长=订购基数）
        → 校验：数量 ≥ 订购下限，数量 ≤ 订购上限，数量 ≤ 今日可用数量
        → 确认 → 调用购物车接口加入购物车
        → Toast 提示「已加入购物车」
```

### 3.5 价格匹配优先级

| 场景 | 大区来源 | 价格组来源 | 说明 |
| --- | --- | --- | --- |
| 未登录 | 系统默认配置 | 系统默认价格组 | 兜底策略，确保访客可浏览商品 |
| 已登录未绑定客户 | 系统默认配置 | 系统默认价格组 | 引导客户先绑定客户资料 |
| 已登录已绑定客户 | 客户所属大区 | 客户绑定的价格组 | 正常业务路径 |

### 4.1 页面整体布局

```
┌─────────────────────────────────────────┐

│          │                              │

│  (选中)   │  │ [主图]  商品名称      │   │
│          │  │        库存:XX  ¥XX   │   │
│  大分类2  │  │         /kg     🛒   │   │
│          │  └──────────────────────┘   │
│  大分类3  │                              │
│          │  ┌──────────────────────┐   │
│  大分类4  │  │ [主图]  商品名称      │   │
│          │  │        库存:XX  ¥XX   │   │
│  大分类5  │  │         /箱     🛒   │   │
│          │  └──────────────────────┘   │
│          │                              │
│          │  ← 滚动到底部自动切下一分类    │

```

**布局说明**：
- 顶部：搜索输入框，固定不随内容滚动
- 左侧：大分类列表，竖向排列，当前选中高亮，可滚动
- 右侧：商品列表（单列卡片），可独立滚动
- 右侧商品滚动到底部后，自动切换到下一个大分类并加载其商品列表

### 4.2 顶部搜索栏

| 控件 | 说明 |
| --- | --- |
| 搜索输入框 | 圆角输入框，占位文案「搜索商品名称」，输入后实时搜索当前分类下商品 |
| 清空按钮 | 输入框右侧，清空后恢复当前分类全部商品 |

**交互逻辑**：
- 搜索范围为当前选中的大分类
- 未选择分类时在全部分类中搜索
- 搜索结果替换右侧商品列表
- 清空搜索词恢复原列表

### 4.3 左侧大分类导航

| 项目 | 说明 |
| --- | --- |
| 数据来源 | 商品分类模块中大分类数据（SAP 同步） |
| 展示内容 | 分类名称（优先使用自定义名称，为空则使用原始名称） |
| 默认选中 | 进入页面默认选中第一个大分类 |
| 选中态 | 高亮背景 + 左侧边线 / 加粗文字 |
| 空状态 | 无分类时展示「暂无分类」 |
| 滚动 | 分类数量多时竖向滚动，不跟手右侧列表 |

### 4.4 右侧商品列表

**商品卡片展示信息**（单列列表，每行一张卡片，左图右文布局）：

| # | 信息项 | 位置 | 说明 |
| --- | --- | --- | --- |
| 1 | 商品主图 | 卡片左侧 | 正方形缩略图，`object-fit: cover`，加载失败展示默认占位图 |
| 2 | 商品名称 | 主图右侧 | 最多 2 行，超出省略号 |
| 3 | 剩余可用库存 | 名称下方 | 格式「库存：XX」，取今日可用数量，≤0 时红色标注「库存不足」 |
| 4 | 价格 | 库存下方 | 较大字号，红色，格式 `¥XX.XX`，来源于价格组明细 |
| 5 | 单位 | 价格同行右侧 | 如 /kg、/箱、/袋，来源于商品资料配置 |
| 6 | 加购按钮 | 卡片右下角 | 购物车图标按钮，点击弹出数量选择器 |

**滚动自动切换分类**：
- 右侧商品列表滚动到底部时，自动切换到下一个大分类
- 自动切换时左侧分类导航同步高亮
- 已是最后一个分类时滚动到底部不继续切换
- 手动点击左侧分类切换不触发自动切换连锁反应（仅手动切换时重置滚动位置）

**排序规则**：
- 按商品条码或后台配置的排序序号展示
- 已上架的商品且今日可用数量 > 0 优先展示（库存不足商品沉底灰显）

**加载方式**：
- 切换分类时一次性加载该分类下全部商品（分页由后端控制，前端滚动加载）

### 4.5 加购数量选择器

```
┌──────────────────────────────────┐
│  选择订购数量                      │
│                                  │
│  商品名称：红富士苹果              │
│  单价：¥8.50 /kg                 │
│                                  │
│  数量：[-] [  10  ] [+]         │
│        （步长=订购基数:10）        │
│                                  │
│  订购范围：10 ~ 100               │
│  可用库存：470                    │
│                                  │
│       [取消]  [加入购物车]        │
└──────────────────────────────────┘
```

**规则**：
- 默认数量 = 订购基数
- 步长 = 订购基数
- 最小值 = 订购下限，最大值 = min(订购上限, 今日可用数量)
- 超出范围时按钮灰显不可点击
- 未登录时点击加购直接跳转登录页，不弹选择器

### 4.6 极限状态

| 状态 | 处理 |
| --- | --- |
| 分类列表为空 | 左侧展示「暂无分类」，右侧为空 |
| 分类下无商品 | 右侧展示「该分类暂无商品」空状态插图 |
| 商品库存为 0 | 卡片沉底灰显，价格划删除线或不展示，加购按钮不可点击 |
| 搜索无结果 | 右侧展示「未找到相关商品，换个关键词试试」 |
| 未登录/未绑定客户 | 使用默认价格组展示价格，加购时引导登录 |
| 价格组下无该商品价格 | 不展示该商品（或展示"价格待定"，后续迭代明确） |
| 网络异常 | 提示「网络异常，请下拉刷新重试」 |
| 图片加载失败 | 展示默认占位图 |

## 5. 数据字典与字段级规则 (Data & Field Rules)

### 5.1 商品分类列表接口

| 接口功能 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 获取大分类列表 | GET | `/api/product/category/major` | 获取所有大分类，含自定义名称 |

**响应字段**：

| 字段名称 | 字段类型 | 来源 | 说明 |
| :--- | :--- | :--- | :--- |
| 分类编码 | String | 商品分类表 | 大分类唯一编码 |
| 分类名称 | String | 商品分类表 | 优先展示自定义名称，为空回退原始名称 |
| 排序序号 | Integer | 商品分类表 | 左侧列表排列顺序 |

### 5.2 分类商品列表接口

| 接口功能 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 分类商品列表 | GET | `/api/product/category/products` | 获取某大分类下的商品列表，含价格和库存 |

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| categoryCode | String | 是 | 大分类编码 |
| keyword | String | 否 | 搜索关键词，模糊匹配商品名称，不传返回全部 |
| regionName | String | 否 | 销售大区，由后端根据客户登录状态自动获取或使用默认值 |
| priceGroupCode | String | 否 | 价格组编码，由后端根据客户登录状态自动获取或使用默认值 |

**响应字段（商品卡片）**：

| 字段名称 | 字段类型 | 来源 | 说明 |
| :--- | :--- | :--- | :--- |
| 商品条码 | String | 商品资料 | SKU 唯一标识 |
| 商品名称 | String | 商品资料 | 展示在卡片上，最多 2 行 |
| 商品主图 | String(URL) | 商品资料 | 卡片图片 |
| 剩余可用库存 | Decimal(10,2) | 商品资料 | 取今日可用数量 |
| 售价 | Decimal(10,2) | 价格组明细 | 按客户价格组匹配，匹配不到则不展示该商品 |
| 单位 | String | 商品资料 | 如 /kg、/箱、/袋 |
| 订购基数 | Decimal(10,2) | 商品资料 | 加购数量步长 |
| 订购下限 | Decimal(10,2) | 商品资料 | 最小订购数量 |
| 订购上限 | Decimal(10,2) | 商品资料 | 最大订购数量 |

### 5.3 价格匹配规则

| 场景 | 大区取值 | 价格组取值 |
| --- | --- | --- |
| 未登录 | 系统配置的默认大区 | 系统配置的默认价格组 |
| 已登录未绑定客户 | 系统配置的默认大区 | 系统配置的默认价格组 |
| 已登录已绑定客户 | 客户绑定的大区 | 客户绑定的价格组编码 |

### 5.4 展示逻辑

| 展示项 | 格式/规则 |
| --- | --- |
| 商品名称 | 最多 2 行，超出省略号（`line-clamp-2`） |
| 价格 | `¥XX.XX`，两位小数，红色字体 |
| 库存 | 格式「库存：XX」，> 0 正常展示，≤ 0 红色「库存不足」 |
| 单位 | 紧邻价格右侧，灰色小字 |
| 商品主图 | 正方形，`object-fit: cover`，加载失败展示占位图 |
| 分类选中态 | 高亮背景色 + 左侧边框色条 |

### 5.5 加购校验规则

- 数量必须 ≥ 订购基数且为订购基数的整数倍
- 数量必须 ≥ 订购下限
- 数量必须 ≤ 订购上限
- 数量必须 ≤ 今日可用数量

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 性能要求

| 指标 | 要求 |
| --- | --- |
| 分类列表接口响应 | < 200ms |
| 商品列表接口响应 | < 800ms |
| 搜索响应 | < 500ms |
| 图片加载 | CDN 加速 + 懒加载 |
| 分类切换 | 商品列表刷新 < 800ms |

### 7.2 权限与安全

| 层级 | 说明 |
| --- | --- |
| 浏览权限 | 无需登录即可浏览默认大区和价格组的商品 |
| 加购权限 | 需登录，未登录时引导跳转登录页 |
| 数据权限 | 客户绑定后仅看到对应大区 + 价格组的商品和价格 |
| 加购频率 | 同一商品 1s 内不可重复加购（防抖） |

### 7.3 业务规则

- 仅展示「已上架」状态的商品
- 价格组下匹配不到售价的商品不展示
- 今日可用数量 ≤ 0 时商品卡片沉底灰显，加购按钮不可点击
- 未登录时使用系统默认大区和默认价格组兜底
- 客户切换绑定客户后，分类页商品列表按新价格组重新加载
- 搜索仅匹配商品名称，在当前选中分类范围内搜索
