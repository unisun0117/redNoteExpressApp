# 商品详情-小程序 SPEC

> **归属中心**：01-商品管理
> **子模块**：商品详情
> **终端**：小程序端
> **版本**：v1.0
> **更新日期**：2026-07-09
>
> **小程序端**：面向 B 端客户，从首页推荐、分类列表或搜索结果点击商品卡片后进入，展示商品的轮播图、价格、库存、订购约束等核心信息，支持数量选择和加入购物车操作。

---

## 1. 背景与目标 (Background & Objectives)

**背景**：客户在浏览商品列表后，需要进入商品详情页获取更全面、完整的商品信息（轮播图、价格、库存、订购约束、商品详情图文等），以便做出下单决策。商品详情页是客户从"浏览"到"加购"的关键转化节点。

**目标**：为小程序客户提供商品详情查看与加购决策所需的全量信息展示，包括轮播图自动轮播、商品基本信息（价格/单位/名称/库存/订购约束）、服务说明、以及后台配置的商品详情富文本内容，并支持选择数量后加入购物车。

**一句话背景**：轮播图 + 商品信息（价格/单位/名称/库存/基数/起订量）+ 服务说明 + 富文本详情，支持数量选择和加购。

## 2. 角色与使用场景 (Roles & Scenarios)

### 目标用户

| 角色 | 说明 |
| --- | --- |
| 访客（未登录/未绑定客户） | 可浏览商品详情（默认大区和默认价格组），点击加购时引导登录 |
| B 端客户（已登录已绑定） | 查看所选客户对应价格组的商品详情，按实际售价选择数量并加入购物车 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 查看商品轮播图 | 进入商品详情页后，顶部轮播图自动轮播展示商品主图和轮播图，3 秒切换一张 |
| 2 | 查看商品基本信息 | 查看商品的当前售价、单位、商品名称、可用库存、订购基数、起订量 |
| 3 | 查看服务信息 | 了解平台提供的配送、售后等服务保障说明 |
| 4 | 查看商品图文详情 | 滚动查看后台商品资料中配置的「商品详情」富文本内容 |
| 5 | 加购商品 | 在底部固定栏选择订购数量（按订购基数步进），点击"加入购物车"完成加购 |
| 6 | 返回列表 | 通过顶部导航栏返回键返回上一页（首页/分类列表/搜索结果） |

## 3. 核心业务流程 (Core Business Flow)

### 3.1 页面加载与数据获取流程

```
客户从首页/分类列表/搜索结果点击商品卡片
    → 进入商品详情页（携带商品条码参数）
    → 判断登录状态
    → 未登录 → 取系统默认配置的大区 + 默认价格组
    → 已登录未绑定客户 → 取系统默认配置的大区 + 默认价格组
    → 已登录已绑定客户 → 取客户绑定的价格组 + 所属大区
    → 请求商品详情数据：
        → 查询商品资料（按销售大区 + 商品条码，状态需为"已上架"）
        → 查询价格组明细（按价格组编码 + 商品条码）获取售价
        → 返回组装后的商品详情数据
    → 渲染页面（轮播图 → 商品信息 → 服务信息 → 商品详情富文本）
```

### 3.2 加购流程

```
在底部固定栏调整订购数量
    → 默认数量 = 订购基数
    → 步长 = 订购基数
    → 最小值 = 订购下限
    → 最大值 = min(订购上限, 今日可用数量)
    → 点击"加入购物车"
    → 未登录 → 引导跳转登录页
    → 已登录 → 调用加购接口
        → 校验订购约束和库存
        → 成功 → Toast "已加入购物车"，购物车角标数量 +1
        → 失败 → Toast 提示具体原因（库存不足/已下架等）
```

### 3.3 价格匹配优先级

| 场景 | 大区来源 | 价格组来源 | 说明 |
| --- | --- | --- | --- |
| 未登录 | 系统默认配置 | 系统默认价格组 | 兜底策略，确保访客可浏览商品详情 |
| 已登录未绑定客户 | 系统默认配置 | 系统默认价格组 | 引导客户先绑定客户资料 |
| 已登录已绑定客户 | 客户所属大区 | 客户绑定的价格组 | 正常业务路径 |

### 4.4 商品信息区域

| # | 信息项 | 位置/样式 | 说明 |
| --- | --- | --- | --- |
| 1 | 售价 | 左上，较大字号（约 24px+），红色加粗 | 格式 `¥XX.XX`，来源于价格组明细。未匹配到价格时展示"价格待定" |
| 2 | 单位 | 售价右侧同行，灰色小字 | 如 `/kg`、`/箱`、`/袋`，来源于商品资料的结算单位 |
| 3 | 商品名称 | 售价下方，加粗，较大字号 | 优先展示小程序名称，为空则使用基础 SKU 表的商品名称 |
| 4 | 今日可用库存 | 名称下方，灰色辅助文字 | 格式「库存：XX」，取今日可用数量；≤0 时红色加粗标注「库存不足」 |
| 5 | 订购基数 | 库存下方，灰色辅助文字 | 格式「订购基数：XX」，客户每次订购的最小基数/步长 |
| 6 | 起订量 | 基数下方，灰色辅助文字 | 格式「起订量：XX」，即订购下限，单次订购的最低数量 |

**展示规则**：
- 价格未匹配（客户价格组下无该商品售价）时，价格显示"价格待定"，加购按钮置灰不可点击
- 库存 ≤ 0 时，"库存：0"红色加粗，加购按钮置灰不可点击
- 商品已下架时，不展示该页面（列表入口已过滤），若通过链接直接进入则提示"商品已下架"

### 4.7 底部固定栏（购物车入口 + 加购操作）

底部固定栏分为左右两部分：左侧为购物车入口（含角标），右侧为加购按钮/数量调整器，根据商品是否已在购物车中切换显示。

#### 4.7.1 商品未加入购物车时

```
┌───────────────────────────────────┐
│  🛒          [    加入购物车    ]  │
│  (角标N)                         │
└───────────────────────────────────┘
```

| 控件 | 类型 | 说明 |
| --- | --- | --- |
| 购物车图标 | 图标按钮 | 左侧，点击跳转至购物车页面 |
| 购物车角标 | 红色数字角标 | 图标右上角，显示购物车中商品明细行数（SKU 种类数，非总件数） |
| 加入购物车 | 按钮 | 右侧主按钮（品牌色/红色），点击触发加购；首次加购数量 = 订购基数 |

#### 4.7.2 商品已加入购物车后

```
┌───────────────────────────────────┐
│  🛒        [ -    10    + ]       │
│  (角标N)                          │
└───────────────────────────────────┘
```

| 控件 | 类型 | 说明 |
| --- | --- | --- |
| 购物车图标 | 图标按钮 | 左侧，点击跳转至购物车页面，角标数字随加购实时更新 |
| 减号按钮 `-` | 按钮 | 数量为订购下限时置灰不可点击；数量减至 0 时从购物车移除，按钮切换回"加入购物车" |
| 数量显示 | 文本 | 显示当前商品在购物车中的已选数量 |
| 加号按钮 `+` | 按钮 | 数量达到 `min(订购上限, 今日可用数量)` 时置灰不可点击 |

#### 4.7.3 按钮切换规则

| 当前状态 | 底部右侧展示 | 说明 |
| --- | --- | --- |
| 商品不在购物车中 | `[加入购物车]` 按钮 | 点击后以订购基数加入购物车，右侧切换为 `[-] N [+]` |
| 商品在购物车中（数量 ≥ 订购下限） | `[-] N [+]` 步进器 | 显示当前购物车中该商品的数量，可加减调整 |
| 步进器减至 0 | 切换回 `[加入购物车]` | 商品从购物车中移除，按钮恢复初始状态 |
| 重新加入购物车 | 切换为 `[-] N [+]` | 默认数量 = 订购基数 |

#### 4.7.4 数量调整规则

- 首次加购数量 = 订购基数
- 步长 = 订购基数（如基数为 10，则数量为 10、20、30...，在购物车已有数量基础上加减）
- 最小值 = 订购下限（减至低于下限时直接移除该商品）
- 最大值 = min(订购上限, 今日可用数量)
- 超出范围时对应按钮灰显不可点击
- 修改数量时实时调用购物车更新接口

#### 4.7.5 购物车角标规则

- 购物车图标右上角始终展示角标数字
- 数字 = 购物车中商品明细行数（不同 SKU 的种类数，非总件数）
- 购物车为空时角标隐藏
- 角标数字 > 99 时展示 "99+"

#### 4.7.6 加购校验

- 未登录 → 引导跳转登录页
- 商品已下架 → 提示"商品已下架，无法购买"，按钮置灰
- 库存不足（今日可用数量 ≤ 0）→ 提示"库存不足"，按钮置灰
- 已加购数量 + 本次增量 > 今日可用数量 → 加号按钮置灰
- 同一商品 1s 内不可重复点击（防抖）

### 4.8 极限状态

| 状态 | 处理 |
| --- | --- |
| 轮播图只有 1 张 | 不自动轮播，不显示指示器，支持手动滑动预览 |
| 轮播图为空 | 展示默认商品占位图（灰色底 + 商品图标） |
| 商品名称过长 | 最多 2 行，超出省略号（`line-clamp-2`） |
| 售价未匹配 | 显示"价格待定"，加购按钮置灰 |
| 库存为 0 | 库存红色标注"库存不足"，加购按钮置灰 |
| 商品已下架 | 列表入口已过滤，直接进入时展示空状态"商品已下架" |
| 商品详情富文本为空 | 隐藏"商品详情"整块区域 |
| 网络加载失败 | Toast "加载失败，请下拉刷新重试" |
| 图片加载失败 | 展示默认占位图 |
| 未登录点击加购 | 引导跳转登录页 |
| 加购成功 | Toast "已加入购物车"，购物车角标 +1 |
| 加购失败 | Toast 提示具体原因（库存不足/网络异常等） |

## 5. 数据字典与字段级规则 (Data & Field Rules)

### 5.1 商品详情接口

| 接口功能 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 商品详情 | GET | `/api/product/detail` | 获取单个商品的完整详情信息（含轮播图、价格、库存、订购约束、商品详情） |

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| barcode | String | 是 | 商品条码 |
| regionName | String | 否 | 销售大区，由后端根据客户登录状态自动获取或使用默认值 |
| priceGroupCode | String | 否 | 价格组编码，由后端根据客户登录状态自动获取或使用默认值 |

**响应字段**：

| 字段名称 | 字段类型 | 来源 | 说明 |
| :--- | :--- | :--- | :--- |
| 商品条码 | String | 商品资料表 | SKU 唯一标识 |
| 商品名称 | String | 基础 SKU 表 | 条码反查的商品名称 |
| 小程序名称 | String | 商品资料表 | 优先展示，为空则使用商品名称 |
| 商品主图 | String(URL) | 商品资料表 | 轮播图第一张（若轮播图为空则用主图） |
| 轮播图列表 | Array\<String(URL)\> | 商品资料表 | 包含主图和轮播图，合并为一个数组。若轮播图为空且主图不为空，则以主图为唯一元素 |
| 售价 | Decimal(10,2) | 价格组明细表 | 按客户价格组匹配售价，匹配不到为 null |
| 结算单位 | String | 商品资料表 | 如 /kg、/箱、/袋 |
| 今日可用数量 | Decimal(10,2) | 商品资料表 | 关联每日商品销售表实时计算（今日可用库存 - 今日已售数量） |
| 订购基数 | Decimal(10,2) | 商品资料表 | 数量调整步长 |
| 订购下限 | Decimal(10,2) | 商品资料表 | 最小订购数量（起订量） |
| 订购上限 | Decimal(10,2) | 商品资料表 | 最大订购数量 |

### 5.3 价格匹配规则

| 场景 | 大区取值 | 价格组取值 |
| --- | --- | --- |

| 已登录未绑定客户 | 系统配置的默认大区 | 系统配置的默认价格组 |
| 已登录已绑定客户 | 客户绑定的大区 | 客户绑定的价格组编码 |

### 5.4 展示逻辑

| 展示项 | 格式/规则 |
| --- | --- |
| 商品名称 | 优先展示小程序名称，为空回退商品名称；最多 2 行，超出省略号 |
| 售价 | `¥XX.XX`，两位小数，红色字体（约 24px+）；未匹配时展示"价格待定" |
| 单位 | 紧邻售价右侧同行，灰色小字 |
| 库存 | `库存：XX`，> 0 正常，≤ 0 红色加粗"库存不足" |
| 订购基数 | `订购基数：XX`，灰色辅助文字 |
| 起订量 | `起订量：XX`，灰色辅助文字 |
| 轮播图 | 满宽，3s 自动轮播，底部圆点指示器 |
| 商品详情 | 使用 `rich-text` 渲染 HTML，为空时隐藏该区域 |

### 5.5 加购校验规则

| 校验项 | 规则 | 不通过处理 |
| --- | --- | --- |
| 登录态 | 需登录 | 引导跳转登录页 |
| 商品状态 | 需为"已上架" | 提示"商品已下架"，按钮置灰 |
| 库存 | 今日可用数量 > 0 | 提示"库存不足"，按钮置灰 |
| 数量下限 | 数量 ≥ 订购下限 | 减至低于下限时从购物车移除商品 |
| 数量上限 | 已加购数量 + 本次增量 ≤ min(订购上限, 今日可用数量) | 加号按钮置灰 |
| 数量步长 | 数量为订购基数的整数倍 | 步进器保证步长正确 |
| 重复提交 | 1s 内不可重复点击 | 按钮防抖 |
| 购物车角标 | 角标 = 购物车中 SKU 种类数（商品明细行数） | 购物车为空时角标隐藏 |

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 性能要求

| 指标 | 要求 |
| --- | --- |
| 商品详情接口响应 | < 500ms |
| 轮播图首图加载 | < 1s（CDN 加速） |
| 富文本渲染 | < 500ms |
| 加购接口响应 | < 500ms |
| 页面首屏加载 | < 1s（含接口请求 + UI 渲染） |
| 图片加载 | CDN 加速 + 懒加载（非首图） |

### 7.2 权限与安全

| 层级 | 说明 |
| --- | --- |
| 浏览权限 | 无需登录即可浏览商品详情（使用默认大区和价格组兜底） |
| 加购权限 | 需登录，未登录时引导跳转登录页 |
| 数据权限 | 客户绑定后仅看到对应大区 + 价格组的商品和价格 |
| 加购频率 | 同一商品 1s 内不可重复加购（防抖） |
| XSS 防护 | 富文本内容渲染前做 XSS 过滤 |

### 7.3 业务规则

- 仅展示"已上架"状态的商品
- 价格组下匹配不到售价时，展示"价格待定"并禁用加购按钮
- 今日可用数量 ≤ 0 时，库存红色标注"库存不足"，加购按钮置灰不可点击
- 数量选择以订购基数为步长，最小值为订购下限，最大值为 min(订购上限, 今日可用数量)
- 未登录时使用系统默认大区和默认价格组兜底展示
- 客户切换绑定客户后回到详情页，价格和库存按新价格组重新加载
- 轮播图仅 1 张时不自动轮播
- 商品详情（富文本）为空时隐藏该区域
