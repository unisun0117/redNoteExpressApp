## 1. 背景与目标 (Background & Objectives)

**背景**：平台面向多个销售大区运营，不同大区可销售的商品范围、价格、库存相互独立。运营管理员需要按「销售大区 + 商品条码」维度维护商品资料，控制商品在小程序端的可见性（上架/下架）、订购约束（基数/下限/上限）以及每日库存。

**目标**：为运营管理员提供商品资料的查询、新增、编辑、导出、导入及库存调整能力，确保每个销售大区下的商品数据准确、可控。

**一句话背景**：按销售大区维度维护商品资料（主键 = 销售大区 + 商品条码），支持查询/新增/编辑/导出/导入/库存调整。

## 2. 角色与使用场景 (Roles & Scenarios)

### 目标用户

| 角色 | 说明 |
| --- | --- |
| 运营管理员 | 负责维护各销售大区下的商品资料，包括上下架、订购约束、库存调整 |
| 区域运营经理 | 按数据权限管理管辖大区内的商品资料 |

### 使用场景（User Story）

| # | 场景 | 描述 |
|---|------|------|
| 1 | 查询商品资料 | 通过商品条码（模糊）、商品名称（模糊）快速检索某大区下的商品资料 |
| 2 | 新增商品资料 | 在指定销售大区下新增商品，关联仓库、设置订购约束和初始库存 |
| 3 | 编辑商品资料 | 修改商品的仓库、状态、小程序名称、图片、详情、订购约束等字段 |
| 4 | 导出商品资料 | 导出商品资料列表（排除图片和富文本字段），用于线下分析或批量整理 |
| 5 | 导入商品资料 | 按导出模板批量导入，系统按「销售大区+商品条码」匹配已有记录执行更新，未匹配到的记录自动新增 |
| 6 | 调整库存 | 在列表直接调整某条商品资料的今日可用库存，系统自动重算今日可用数量 |

## 3. 核心业务流程 (Core Business Flow)

### 3.1 商品资料查询流程

```
进入商品资料页面
    → 可选选择销售大区、输入商品条码（模糊）、商品名称（模糊）、选择状态
    → 点击"查询"
    → 列表刷新展示符合条件的商品资料（分页）
```

### 3.2 新增商品资料流程

```
点击"新增"按钮
    → 进入新增子页面
    → 选择销售大区（必选、单选，下拉数据来源于销售大区管理模块）
    → 输入商品条码（必填），系统自动反查商品名称（来源于底层商品条码表）
    → 选择仓库（单选，下拉选项联动销售大区，数据来源于仓库信息管理模块）
    → 状态默认"已上架"
    → 选填小程序名称
    → 输入订购基数、订购下限、订购上限（均必填，最多两位小数）
    → 选填商品主图（一张）、轮播图（最多9张）、商品详情（富文本）
    → 输入每日可用库存（必填，最多两位小数）
    → 今日可用数量默认取每日可用库存值，今日已售数量默认 0
    → 提交保存，校验「销售大区 + 商品条码」唯一性
    → 保存成功，返回列表页并刷新
```

### 3.3 编辑商品资料流程

```
在列表中点击目标行的"编辑"按钮
    → 进入编辑子页面，回填当前数据
    → 不可修改字段（灰显只读）：销售大区、商品条码、商品名称、每日可用库存、今日可用数量、今日已售数量
    → 可修改字段：仓库、状态、小程序名称、商品主图、轮播图、商品详情、订购基数、订购下限、订购上限
    → 提交保存
    → 保存成功，返回列表页并刷新
```

### 3.4 导出流程

```
点击"导出"按钮
    → 系统按当前查询条件导出 Excel
    → 排除字段：商品主图、轮播图、商品详情（富文本）
    → 其余列表字段全部导出
```

### 3.5 导入流程

```
点击"导入"按钮
    → 上传按导出模板格式填写的 Excel
    → 系统按「销售大区 + 商品条码」逐行匹配
    → 匹配到已有记录 → 更新以下字段：仓库、状态、小程序名称、订购基数、订购下限、订购上限
    → 未匹配到记录 → 作为新记录新增（必填字段同新增规则，默认值同新增规则）
    → 校验通过后批量执行，返回导入结果（新增 N 条 / 更新 N 条 / 失败 N 条 / 失败原因）
```

### 3.6 库存调整流程

```
在列表中点击目标行的"调整库存"按钮
    → 弹出库存调整弹窗
    → 展示只读字段：销售大区、商品条码、商品名称
    → 今日可用库存（可编辑修改）
    → 今日可用数量 = 修改后的今日可用库存 - 今日已售数量（系统自动计算，不可编辑）
    → 今日已售数量（不可编辑）
    → 确认保存，刷新列表
```

│                                             │

**规则**：
- 销售大区、商品条码、商品名称均为只读展示
- 今日可用库存可编辑修改，修改后今日可用数量自动重算
- 计算公式：今日可用数量 = 今日可用库存 - 今日已售数量
- 今日已售数量不可编辑，由系统订单侧实时同步

| 状态 | 处理 |
| --- | --- |

| 富文本内容过长 | 编辑页面中不做截断限制，由后端存储容量控制 |

### 5.2 新增逻辑

- 「销售大区」下拉选项来源于销售大区管理模块
- 「商品条码」输入后触发商品名称反查（关联基础SKU表），反查失败提示"未找到该商品条码"
- 「仓库」下拉选项联动销售大区，仅展示所选大区下的仓库，提交时存储仓库ID（`warehouse_id`）
- 「状态」默认"已上架"
- 提交时校验「销售大区 + 商品条码」唯一性，重复则拒绝并提示
- 商品名称（基础SKU表）、仓库编码/名称（仓库基础表）、今日已售/可用数量（每日商品销售表）不在本表存储，列表/详情接口返回时由 Service 层关联查询填充

### 5.3 编辑逻辑

- **不可修改字段**（灰显只读）：销售大区、商品条码、每日可用库存
- **可修改字段**：仓库（存储仓库ID）、状态、小程序名称、商品主图、轮播图、商品详情、订购基数、订购下限、订购上限
- 仓库下拉仍联动销售大区（虽然大区不可修改，但仓库过滤逻辑保持一致）
- 商品名称、仓库编码/名称、今日可用/已售数量为关联查询字段，不在编辑范围内

### 5.4 导出逻辑

| 是否导出 | 字段 |
|:---|:---|
| ✅ 导出 | 销售大区、商品条码、商品名称（关联基础SKU表）、仓库编码（关联仓库基础表）、仓库名称（关联仓库基础表）、状态、小程序名称、订购基数、订购下限、订购上限、每日可用库存、今日可用数量（关联每日商品销售表计算）、今日已售数量（关联每日商品销售表） |
| ❌ 不导出 | 仓库ID（内部键值）、商品主图、轮播图、商品详情（富文本） |

### 5.5 导入逻辑

- 按模板 Excel 导入，支持新增和更新两种模式
- 匹配键：「销售大区 + 商品条码」
- **匹配到已有记录 → 更新**，可更新字段：仓库（Excel中的仓库编码解析为仓库ID）、状态、小程序名称、订购基数、订购下限、订购上限
- **未匹配到记录 → 新增**，必填字段同新增规则（销售大区、商品条码、仓库编码（解析为仓库ID）、订购基数/下限/上限、每日可用库存），默认值规则同新增（状态默认已上架），商品名称由条码反查基础SKU表
- **不可通过导入修改的字段**：销售大区（新增时可输入）、商品条码（新增时可输入）、每日可用库存（仅新增时可输入，更新时忽略）
- **导入时忽略的参考字段**：商品名称、仓库名称、今日可用数量、今日已售数量（这些字段由关联表查询，导入时不做处理）
- 导入时仓库编码通过仓库基础表解析为仓库ID存储
- 导入完成后展示结果汇总（新增 N 条 / 更新 N 条 / 失败 N 条 / 失败原因列表）

### 5.6 库存调整逻辑

- 仅通过列表「调整库存」按钮触发，弹窗操作
- 展示只读：销售大区、商品条码、商品名称（关联基础SKU表）
- 「每日可用库存」可编辑修改（替代编辑页面中该字段只读的限制，仅更新 `prd_product_item.daily_available_stock`）
- 「今日已售数量」由 Service 层关联每日商品销售表实时查询（不可编辑）
- 「今日可用数量」= 每日可用库存 − 今日已售数量（系统自动计算，不可手动编辑）
- 保存后刷新列表当前行数据

### 5.7 展示逻辑

- 状态列：已上架（绿色标签）/ 已下架（灰色标签）
- 商品主图、轮播图、商品详情不在列表中展示，仅在新增/编辑子页面中维护和预览
- 日期时间格式统一为 `YYYY-MM-DD HH:mm:ss`
- 小数位数字段保留 2 位小数展示

## 6. 系统交互与边界 (System Integrations & Boundaries)

### 6.1 前置依赖

| 依赖系统/模块 | 依赖内容 | 说明 |
| :--- | :--- | :--- |
| 销售大区管理（07-运营管理） | 销售大区下拉数据源 | 新增/编辑时选择大区，列表展示大区名称 |
| 仓库信息管理（07-运营管理） | 仓库下拉数据源（含仓库ID、编码、名称） | 选择仓库时联动大区过滤，本表存储仓库ID；列表/导出时通过仓库ID关联查询编码和名称 |
| 基础SKU表 | 商品条码 → 商品名称反查 | 输入条码后自动查询商品名称；列表/导出时通过条码批量关联查询 |
| 每日商品销售表 | 今日已售数量查询 | 列表/详情/导出时通过 region_code + barcode + 日期关联查询；今日可用数量由此计算 |
| SAP 系统 | 仓库编码同步 | 仓库基础表的编码来源于 SAP，平台不自行创建 |

### 6.2 上下游影响

| 方向 | 模块 | 影响 |
| :--- | :--- | :--- |
| 提供 → | 商品搜索（小程序） | 小程序端商品列表按上架状态和销售大区过滤 |
| 提供 → | 商品分类页面（小程序） | 按分类浏览时展示对应大区下的已上架商品 |
| 提供 → | 购物车/下单（小程序） | 下单时校验订购基数/下限/上限，扣减库存 |
| 提供 → | 商品价格管理 | 价格按仓库 + 商品维度维护，依赖本模块的商品资料 |
| 依赖 ← | 销售大区管理 | 大区下拉选项数据源 |
| 依赖 ← | 仓库信息管理 | 仓库下拉选项数据源 |
| 依赖 ← | 底层商品条码表 | 商品条码反查名称 |

### 6.3 库存数据流

```
prd_product_item.daily_available_stock（平台维护/导入/库存调整）
    → 每日商品销售表.today_sold_qty（订单系统实时同步）
        → 今日可用数量 = 每日可用库存 - 今日已售数量（列表/详情接口返回时实时计算）
            → 小程序下单时校验今日可用数量 ≥ 订购数量

今日已售数量 ← 订单系统实时同步至每日商品销售表（下单成功后累加）
今日可用数量 ← 不存储，列表/详情/导出接口返回时实时计算
```

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 权限与安全

| 维度 | 要求 |
| :--- | :--- |
| 查看权限 | 按用户数据权限过滤销售大区，用户仅能看到管辖大区内的商品资料 |
| 编辑权限 | 仅具备运营管理权限的用户可新增、编辑、导入、库存调整 |
| 导出权限 | 具备查看权限即可导出 |
| 数据安全 | 商品主图、轮播图上传需校验文件类型和大小，防止恶意文件上传 |

### 7.2 性能要求

| 维度 | 要求 |
| :--- | :--- |
| 列表加载 | 支持分页，默认每页 20 条 |
| 查询响应 | 模糊查询响应时间 < 2s |
| 新增/编辑保存 | 保存操作即时反馈，图片异步上传 |
| 导出 | 单次导出上限 10,000 条，超出时分批导出或提示缩小筛选范围 |
| 导入 | 单次导入上限 1,000 条，超出提示分批导入 |

### 7.3 业务规则

- 主键为「销售大区 + 商品条码」，联合唯一
- 同一销售大区下，同一商品条码只能有一条商品资料记录
- 商品条码输入后反查商品名称，反查失败不允许提交
- 商品下架后小程序端不可见，但不影响已有订单
- 订购基数 ≤ 订购下限 ≤ 订购上限，提交时校验三者逻辑关系
- 每日可用库存仅通过「库存调整」功能修改，编辑页面不可修改
- 导入匹配到已有记录时更新指定字段（不清空未包含字段），未匹配到时自动新增
- 库存调整的今日可用数量 = 今日可用库存 - 今日已售数量，结果不能为负数
