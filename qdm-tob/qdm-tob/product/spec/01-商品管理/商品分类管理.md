> **更新日期**：2026-07-03

---

## 1. 背景与目标 (Background & Objectives)

**背景**：商品的三级分类数据（大分类、中分类、小分类）来源于外部 SAP 系统，平台仅做展示同步和自定义名称维护，不在本平台内新增、删除分类。平台用户在查看商品或配置商品相关业务时，需要使用分类数据进行筛选或引用。

**目标**：为运营管理员提供商品分类数据的查询和自定义名称编辑能力，确保平台侧可对 SAP 同步的分类进行本地化命名，方便业务人员识别和使用。

**一句话背景**：展示 SAP 同步的三级商品分类数据，支持按分类级别和名称查询，允许编辑自定义名称。

## 3. 核心业务流程 (Core Business Flow)

### 3.1 整体数据流

```
SAP 系统 ──同步──→ 商品分类表 ──展示──→ 平台后台（商品分类管理页面）
                         │
                         └──编辑──→ 自定义名称（平台侧唯一可修改字段）
```

### 3.2 分类层级关系

```
大分类（一级）
  └── 中分类（二级）
        └── 小分类（三级）
```

三级分类数据存放在同一张表，通过"上级编码"字段关联父子关系。

### 3.3 查询流程

1. 进入商品分类管理页面
2. 在筛选区选择分类级别（全部 / 大分类 / 中分类 / 小分类），或输入分类编码、分类名称进行模糊查询
3. 点击"查询"，列表刷新展示符合条件的分类数据

### 3.4 编辑自定义名称流程

1. 在列表中定位目标分类行
2. 点击行末的"编辑"按钮
3. 弹出编辑弹窗，展示分类编码（只读）、分类名称（只读）、自定义名称（可编辑）
4. 修改自定义名称（可为空），点击"保存"
5. 系统更新记录并刷新列表

## 5. 数据字典与字段级规则 (Data & Field Rules)

### 5.1 商品分类表核心字段

| 字段名称 | 字段类型 | 来源/依赖 | 默认值 | 读写权限 | 校验规则与约束 | 说明/占位符 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 分类级别 | Enum | SAP 同步 | — | 只读 | 枚举：大分类 / 中分类 / 小分类 | 由 SAP 数据决定 |
| 分类编码 | String(60) | SAP 同步 | — | 只读 | 全局唯一 | SAP 原始编码，不可修改 |
| 分类名称 | String(100) | SAP 同步 | — | 只读 | SAP 原始名称 | SAP 原始名称，不可修改 |
| 自定义名称 | String(100) | 平台编辑 | — | 可编辑 | 可为空，最长 100 字符 | 平台侧唯一可修改字段；为空时展示原始分类名称 |
| 上级编码 | String(60) | SAP 同步 | — | 只读 | 关联父级分类编码 | 大分类的上级编码为空 |
| 上级名称 | String(100) | SAP 同步 | — | 只读 | — | 大分类的上级名称为空 |
| 创建时间 | DateTime | 系统记录 | 同步时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | SAP 数据首次同步时间 |
| 更新时间 | DateTime | 系统记录 | 同步时间 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 自定义名称修改或 SAP 数据更新时刷新 |

### 5.2 展示逻辑

- 分类级别展示：大分类（蓝色标签）/ 中分类（绿色标签）/ 小分类（灰色标签）
- 上级编码/上级名称：大分类行展示"-"
- 自定义名称为空时，列表列展示"-"，前端使用时回退展示原始分类名称
- 日期时间格式统一为 `YYYY-MM-DD HH:mm:ss`

### 5.3 编辑逻辑

- 仅"自定义名称"字段可编辑，其他字段均只读
- 自定义名称可为空（清空后保存），保存后列表对应列显示"-"
- 编辑保存后仅刷新当前行数据，不影响列表其他行

## 6. 系统交互与边界 (System Integrations & Boundaries)

### 6.1 前置依赖

| 依赖系统 | 依赖内容 | 说明 |
| :--- | :--- | :--- |
| SAP 系统 | 商品三级分类数据 | 分类数据由 SAP 定时或实时同步至平台，平台不负责数据的创建和删除 |

### 6.2 上下游影响

| 方向 | 模块 | 影响 |
| :--- | :--- | :--- |
| 提供 → | 商品资料管理 | 商品资料中的分类字段引用本模块的分类编码和名称 |
| 提供 → | 商品查询/筛选 | 前端分类筛选器的数据来源 |
| 提供 → | 其他需引用分类的业务模块 | 统一通过本模块的分类编码和自定义名称进行展示 |
| 依赖 ← | SAP 系统 | 商品分类全部数据来源于 SAP，SAP 数据变更后同步至本模块 |

### 6.3 数据同步

- 分类数据由 SAP 系统同步写入，平台不开放新增和删除功能
- SAP 新增、修改、删除分类时，平台同步更新对应记录
- SAP 删除分类时，若该分类已被商品引用，需在商品侧处理好引用关系后方可删除

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 权限与安全

| 维度 | 要求 |
| :--- | :--- |
| 查看权限 | 所有登录用户可查看分类列表 |
| 编辑权限 | 仅具备运营管理权限的用户可编辑自定义名称 |
| 数据安全 | 分类编码和名称由 SAP 同步，平台不可篡改 |

### 7.2 性能要求

| 维度 | 要求 |
| :--- | :--- |
| 列表加载 | 支持分页，单页数据量不超过 50 条 |
| 查询响应 | 模糊查询响应时间 < 2s |
| 编辑保存 | 保存操作即时反馈 |

### 7.3 业务规则

- 分类数据全部来源于 SAP，平台不允许新增、删除分类
- 分类编码全局唯一，为 SAP 原始编码，不可修改
- 仅"自定义名称"字段允许平台侧编辑，用于本地化命名
- 自定义名称为空时，系统各处展示回退使用原始分类名称
- 三级分类通过"上级编码"字段关联（大分类上级编码为空）
- SAP 侧分类编码变更时，平台需同步更新关联商品的分类引用
