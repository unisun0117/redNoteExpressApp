> **归属中心**：01-商品管理
> **模块**：价格审批记录
> **版本**：v1.1
> **更新日期**：2026-07-09

---

## 1. 背景与目标 (Background & Objectives)

**背景**：商品价格管理模块中，当运营管理员编辑售价且变动比率超过配置阈值时，系统生成待审批记录。审批通过后售价生效，审批驳回则售价恢复原值。价格审批记录模块集中展示所有价格调整的审批历史，供业务人员查询和追溯。

**目标**：为业务人员提供价格审批记录的多维度检索和列表查看能力，支持按仓库、价格组、商品、状态、申请时间等条件组合筛选，便于审计和业务追溯。

**一句话背景**：集中展示商品价格调整的审批历史记录，支持多维度检索查询。

## 3. 核心业务流程 (Core Business Flow)

### 3.1 审批记录查询流程

```
进入价格审批记录页面
    → 可选选择仓库名称、价格组、输入商品信息、选择状态、选择申请时间范围
    → 点击"查询"
    → 列表刷新展示符合条件的审批记录（分页）
```

### 3.2 审批记录来源

```
商品价格管理 → 编辑售价 → 变动比率 > 阈值 → 生成待审批记录
    → 提交审批
        → 通过 → 记录状态 = "已通过"，售价生效
        → 驳回 → 记录状态 = "已驳回"，售价恢复原值
    → 所有审批记录汇总至本页面展示
```

### 4.4 展示逻辑

- **序号**：前端自动生成，从 1 开始递增，跨页累计
- **价格字段**：保留 2 位小数，格式 `¥XX.XX`
- **价格差**：正值红色 `+XX.XX`，负值绿色 `-XX.XX`，零值灰色 `0.00`
- **价格变动比率**：格式 `+XX.XX%` 或 `-XX.XX%`，正值红色、负值绿色，与价格差同色系；超过阈值 30% 时加粗标注
- **状态**：待审批（橙色标签）、已通过（绿色标签）、已驳回（红色标签）
- **调整理由**：超长时省略号截断，鼠标悬浮 Tooltip 展示全文
- **申请时间**：格式 `YYYY-MM-DD HH:mm:ss`
- **待办人**：多人时以逗号分隔展示，超长时省略号截断 + Tooltip 展示完整列表；未分配时展示 `-`
- **审批人**：未审批时展示 `-`，审批后展示审批操作人姓名
- **审批时间**：未审批时展示 `-`，审批后展示审批完成时间

### 4.5 极限状态

| 状态 | 处理 |
| --- | --- |
| 列表为空 | 展示「暂无审批记录」空状态占位图 |
| 无查询结果 | 展示「未找到匹配的审批记录」 |
| 数据极多 | 列表分页展示，默认每页 20 条 |
| 日期范围无效 | 开始日期 > 结束日期时提示「开始日期不能晚于结束日期」 |
| 价格差为 0 | 正常展示 `0.00`，灰色字体 |

| 字段名称 | 字段类型 | 来源/依赖 | 读写权限 | 校验规则与约束 | 说明/占位符 |

| 价格组名称 | String | 关联价格组表 | 只读 | — | 审批记录关联的价格组 |

| 调整理由 | String(200) | 价格组明细编辑 | 只读 | 来源于编辑售价时填写的变动原因 | 审批发起时快照保存 |
| 申请人 | String | 系统记录 | 只读 | 当前登录用户 | 发起审批申请的操作人 |

| 审批人 | String | 审批流程 | 只读 | 审批操作人（实际执行审批的那一个人） | 未审批时为空 |
| 审批时间 | DateTime | 审批流程 | 只读 | 审批操作时间 | 未审批时为空 |
| 申请时间 | DateTime | 系统记录 | 只读 | 格式 YYYY-MM-DD HH:mm:ss | 发起审批申请的时间 |

### 5.2 审批记录生命周期

```
编辑售价 → 变动比率 > 阈值
    → 生成审批记录（状态=待审批，变更前价格=原售价快照，变更后价格=新售价）
    → 提交审批
        → 通过 → 状态=已通过，价格组明细售价更新为变更后价格
        → 驳回 → 状态=已驳回，价格组明细售价保持变更前价格不变
```

### 5.3 查询逻辑

- 销售大区：下拉联动数据来源于销售大区管理模块
- 价格组：下拉联动数据来源于价格组管理模块
- 商品信息：模糊匹配商品条码或商品名称，输入时同时按条码和名称搜索
- 状态：下拉单选，可清空（清空表示查询全部状态）
- 申请时间：页面加载默认近七天，支持快捷选项切换或手动自定义日期范围，开始日期 ≤ 记录申请时间 ≤ 结束日期

## 7. 非功能性需求 (Non-Functional Requirements)

### 7.1 权限与安全

| 维度 | 要求 |
| :--- | :--- |
| 查看权限 | 按用户数据权限过滤销售大区，用户仅能看到管辖范围内的审批记录 |
| 操作权限 | 本页面仅提供查看功能，审批操作在审批管理模块完成 |
| 数据安全 | 审批记录为只读数据，不可编辑或删除 |

### 7.2 性能要求

| 维度 | 要求 |
| :--- | :--- |
| 列表加载 | 支持分页，默认每页 20 条 |
| 查询响应 | 组合条件查询响应时间 < 2s |
| 日期范围查询 | 大数据量时间段查询建议限定范围不超过 90 天 |

### 7.3 业务规则

- 审批记录由价格管理模块编辑售价超阈值时自动生成，不可手动新增
- 审批记录为只读，不可编辑或删除
- 价格差 = 变更后价格 - 变更前价格，正值表示涨价，负值表示降价
- 列表按申请时间倒序排列（最新记录在前）
- 状态为"待审批"的记录，实际售价仍为变更前价格
- 审批通过后，价格组明细中的售价更新为变更后价格
- 审批驳回后，价格组明细中的售价保持变更前价格不变
