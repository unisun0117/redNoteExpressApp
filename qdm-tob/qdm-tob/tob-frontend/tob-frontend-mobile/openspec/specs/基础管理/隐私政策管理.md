# 小程序隐私文档维护 PRD

> **归属中心**：06-基础管理
> **子模块**：隐私文档维护
> **终端**：后台管理端（Web）
> **版本**：v1.0
> **更新日期**：2026-07-07

------

## 1. 需求概述

### 1.1 模块定位

运营管理员在后台统一维护 6 种法律合规文档（隐私政策、用户协议等），通过「未发布 → 已发布 → 已下架」三态流转控制前台（小程序端）可见性。其中**隐私政策**具备多版本管理、PDF 上传等扩展能力，其余 5 种类型遵循统一规则。

### 1.2 核心规则总览

| 规则项 | 隐私政策 | 其他 5 种类型 |
|--------|:-------:|:-----------:|
| 三态流转（未发布→已发布→已下架） | ✅ | ✅ |
| 编辑保存后状态回退为「未发布」 | ✅ | ✅ |
| 多版本管理（同类型仅一个已发布版本） | ✅ | ❌ 单版本 |
| 版本号字段（必填，不可重复） | ✅ | ❌ 无此字段 |
| PDF 上传（必填，≤10MB） | ✅ | ❌ 无此功能 |
| 列表「下载文件」按钮 | ✅ | ❌ 无此按钮 |

### 1.3 文档类型

| 序号 | 类型名称 |
|------|---------|
| 1 | 隐私政策 |
| 2 | 隐私政策摘要 |
| 3 | 用户管理规则及公约 |
| 4 | 个人信息收集清单 |
| 5 | 第三方共享个人信息清单 |
| 6 | 用户协议 |

## 2. 状态流转

### 2.1 流转规则（统一适用于全部 6 种类型）

```
新建记录 ──→ 未发布 ──点击发布──→ 已发布 ──点击下架──→ 已下架（终态）
                ↑                    │                      │
                │                    │                      │
                └── 编辑保存 ────────┘──────────────────────┘
                  （任意状态编辑后统一回退）
```

| 当前状态 | 可执行操作 | 目标状态 | 说明 |
|----------|-----------|----------|------|
| 未发布 | 发布 | 已发布 | 前台可见 |
| 未发布 | 编辑保存 | 未发布 | 仍是未发布 |
| 已发布 | 下架 | 已下架 | 前台不可见，不可逆 |
| 已发布 | 编辑保存 | 未发布 | 状态自动回退 |
| 已下架 | 编辑保存 | 未发布 | 状态自动回退 |

关键约束：

- **新建记录默认状态为「未发布」**，前台不可见。
- **已下架为终态**，不可逆，不再提供发布或下架按钮。
- **编辑任意状态的记录并保存后，状态统一回退为「未发布」**。

### 2.2 隐私政策发布特殊逻辑

隐私政策发布时，系统自动执行：

1. 查询当前「隐私政策」类型下状态为「已发布」的所有记录
2. 将这些记录状态批量更新为「已下架」
3. 将当前记录状态更新为「已发布」

以上三步需在同一事务中完成，保证**同一时间仅一个隐私政策版本处于已发布状态**。

## 3. 列表页

### 3.1 筛选区

| 筛选项 | 组件 | 说明 |
|--------|------|------|
| 搜索框 | 文本输入框 | 按 **H5 地址** 或 **备注** 字段模糊搜索 |
| 类型下拉 | 下拉选择器 | 选项：全部 + 6 种文档类型，精确匹配 |
| 版本号 | 文本输入框 | 精确匹配，**仅对「隐私政策」类型生效**，筛选其他类型时此条件自动忽略 |

### 3.2 列表字段

| 列名 | 说明 |
|------|------|
| 序号 | 自增编号 |
| 状态 | 「未发布」「已发布」「已下架」，建议用不同颜色标签区分（灰/绿/红） |
| 类型 | 6 种文档类型名称 |
| H5 地址 | 完整 URL，过长时截断显示 |
| 版本号 | 隐私政策显示版本号（如 `2.5`），**其他 5 种类型显示 `-`** |
| 备注 | 过长时截断显示 |
| 创建人 | 创建该记录的用户 |
| 创建时间 | 格式 `YYYY-MM-DD HH:mm` |
| 操作 | 按钮组，按状态和类型动态显隐（见 §3.3） |

### 3.3 操作栏按钮显隐

| 状态 | 隐私政策 | 其他 5 种类型 |
|------|---------|-------------|
| 未发布 | 【复制地址】【详情】【发布】**【下载文件】** | 【复制地址】【详情】【发布】 |
| 已发布 | 【复制地址】【详情】【下架】**【下载文件】** | 【复制地址】【详情】【下架】 |
| 已下架 | 【复制地址】【详情】**【下载文件】** | 【复制地址】【详情】 |

### 3.4 按钮交互

| 按钮 | 行为 |
|------|------|
| 复制地址 | 将 H5 地址写入剪贴板，Toast 提示「复制成功」 |
| 详情 | 进入编辑页，回显全部字段（文档类型禁止修改） |
| 发布 | 二次确认弹窗 → 状态变更为「已发布」。隐私政策额外自动下架同类型旧版本 |
| 下架 | 二次确认弹窗 → 状态变更为「已下架」（不可逆） |
| 下载文件 | 下载关联的 PDF 文件到本地（仅隐私政策） |

### 3.5 排序与分页

- **默认排序**：按创建时间倒序（最新在前）
- **分页**：每页 10 条，可选 10 / 20 / 50
- **分页组件**：列表底部居中

## 4. 新增/编辑表单

### 4.1 表单字段

| 字段 | 组件 | 必填 | 说明 |
|------|------|:--:|------|
| 文档类型 | 下拉选择 | ✅ | 6 种类型之一；**编辑时禁用修改** |
| 版本号 | 文本输入 | 条件 | **仅「隐私政策」显示且必填**；仅允许数字和点号（如 `2.5`）；同一类型下不可重复；其他类型隐藏 |
| 上传文件 | 文件上传 | 条件 | **仅「隐私政策」显示且必填**；仅 `.pdf` 格式，单文件 ≤ 10MB；其他类型隐藏 |
| URL 地址 | 文本输入 | ✅ | 必须是合法的 `http://` 或 `https://` 链接 |
| 备注 | 多行文本域 | ❌ | 最多 200 字 |

### 4.2 字段显隐逻辑

```
if (文档类型 === '隐私政策') {
  显示：版本号（必填）
  显示：上传文件（必填，PDF ≤ 10MB）
} else {
  隐藏：版本号
  隐藏：上传文件
}
```

### 4.3 校验规则

| 校验项 | 触发时机 | 规则 | 提示文案 |
|--------|---------|------|---------|
| 版本号格式 | 失焦 | 仅数字和点号 |「版本号格式不正确，仅允许数字和点号」 |
| 版本号唯一 | 失焦 | 同类型下不重复 |「该版本号已存在」 |
| URL 格式 | 失焦 | `http://` 或 `https://` 开头 |「请输入合法的链接地址」 |
| 文件格式 | 选择文件 | 仅 `.pdf` |「仅支持 PDF 格式」 |
| 文件大小 | 选择文件 | ≤ 10MB |「文件大小不能超过 10MB」 |
| 备注字数 | 输入 | ≤ 200 字 | 底部实时显示剩余可输入字数 |

### 4.4 保存行为

- 新增：保存后状态为「未发布」
- 编辑：保存后状态**自动回退为「未发布」**（无论之前是什么状态）
- 保存成功后关闭表单，列表自动刷新

## 5. 数据字典

### 5.1 数据库表结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| id | BIGINT | ✅ | 主键，自增 |
| doc_type | VARCHAR(50) | ✅ | 文档类型枚举 |
| version | VARCHAR(20) | ❌ | 版本号，仅隐私政策有值 |
| h5_url | VARCHAR(500) | ✅ | H5 页面链接 |
| file_url | VARCHAR(500) | ❌ | PDF 文件地址，仅隐私政策有值 |
| remark | VARCHAR(200) | ❌ | 备注 |
| status | VARCHAR(20) | ✅ | `UNPUBLISHED` / `PUBLISHED` / `ARCHIVED` |
| created_by | VARCHAR(50) | ✅ | 创建人 |
| created_at | DATETIME | ✅ | 创建时间 |
| updated_by | VARCHAR(50) | ❌ | 最后更新人 |
| updated_at | DATETIME | ❌ | 最后更新时间 |

### 5.2 枚举定义

**文档类型枚举**：

| 枚举值 | 显示名称 |
|--------|---------|
| `PRIVACY_POLICY` | 隐私政策 |
| `PRIVACY_SUMMARY` | 隐私政策摘要 |
| `USER_RULES` | 用户管理规则及公约 |
| `INFO_COLLECTION` | 个人信息收集清单 |
| `THIRD_PARTY_SHARING` | 第三方共享个人信息清单 |
| `USER_AGREEMENT` | 用户协议 |

**状态枚举**：

| 枚举值 | 显示名称 | 前台可见 | 是否为终态 |
|--------|---------|:------:|:--------:|
| `UNPUBLISHED` | 未发布 | ❌ | ❌ |
| `PUBLISHED` | 已发布 | ✅ | ❌ |
| `ARCHIVED` | 已下架 | ❌ | ✅ |

### 5.3 索引建议

| 索引 | 字段 | 说明 |
|------|------|------|
| 联合唯一索引 | `(doc_type, version)` | 保证同类型下版本号不重复（version 为 NULL 时不受约束） |

## 6. API 接口

### 6.1 列表查询

```
GET /api/privacy/docs
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| keyword | String | ❌ | H5 地址或备注模糊搜索 |
| docType | String | ❌ | 类型枚举值，空=全部 |
| version | String | ❌ | 版本号精确匹配（仅隐私政策有效） |
| pageNum | Integer | ✅ | 页码 |
| pageSize | Integer | ✅ | 每页条数 |

**响应**：

```json
{
  "total": 25,
  "pageNum": 1,
  "pageSize": 10,
  "list": [{
    "id": 1,
    "docType": "PRIVACY_POLICY",
    "docTypeName": "隐私政策",
    "version": "2.5",
    "h5Url": "https://xxx.com/privacy",
    "fileUrl": "https://cdn.xxx.com/privacy-v2.5.pdf",
    "remark": "2026年7月更新",
    "status": "PUBLISHED",
    "statusName": "已发布",
    "createdBy": "张三",
    "createdAt": "2026-07-01 10:00:00",
    "updatedBy": "张三",
    "updatedAt": "2026-07-05 14:20:00"
  }]
}
```

### 6.2 新增

```
POST /api/privacy/docs
```

请求体：`docType`（必填）、`version`（条件）、`h5Url`（必填）、`fileUrl`（条件）、`remark`（选填）。
保存后状态自动设为 `UNPUBLISHED`。

### 6.3 编辑

```
PUT /api/privacy/docs/{id}
```

请求体同新增，`docType` 不可修改。保存后状态自动回退为 `UNPUBLISHED`。

### 6.4 发布

```
PUT /api/privacy/docs/{id}/publish
```

状态 `UNPUBLISHED` → `PUBLISHED`。若类型为隐私政策，事务内先下架同类型所有已发布版本，再发布当前版本。

### 6.5 下架

```
PUT /api/privacy/docs/{id}/archive
```

状态 `PUBLISHED` → `ARCHIVED`。终态操作，不可逆。

### 6.6 版本号校验

```
GET /api/privacy/docs/check-version?docType=PRIVACY_POLICY&version=2.5&excludeId=1
```

响应：`{ "exists": true }` 或 `{ "exists": false }`

### 6.7 文件上传

```
POST /api/common/upload
Content-Type: multipart/form-data
```

校验：仅 `.pdf`，≤ 10MB。返回：`{ "url": "https://cdn.xxx.com/files/xxx.pdf" }`

## 7. 非功能性需求

### 7.1 性能

| 指标 | 要求 |
|------|------|
| 列表查询 | < 500ms |
| 文件上传（10MB） | < 3s |
| 版本号唯一性校验 | < 200ms |

### 7.2 权限

| 层级 | 说明 |
|------|------|
| 菜单可见 | 仅「运营管理员」角色 |
| 操作权限 | 新增、编辑、发布、下架分别控制 |

### 7.3 异常处理

| 场景 | 处理 |
|------|------|
| 文件非 PDF | 前端拦截 + Toast「仅支持 PDF 格式」 |
| 文件超 10MB | 前端拦截 + Toast「文件大小不能超过 10MB」 |
| URL 格式非法 | 前端校验 + Toast「请输入合法的链接地址」 |
| 版本号重复 | 后端校验 + Toast「该版本号已存在」 |
| 发布事务失败 | 回滚 + Toast「发布失败，请重试」 |
