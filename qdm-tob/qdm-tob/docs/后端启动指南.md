# 后端启动指南（tob-backend）

> 适用人群：产品、开发、测试——任何需要在本机把后端跑起来的人。
> 读完后照做即可，不需要懂 Maven 细节。文中的路径仅为示例，可按你自己的习惯调整。

---

## 一、它是什么

`tob-backend` 是一个 Java 后端服务：

- **技术栈**：Java 25 + Spring Boot 4.0.7 + MyBatis-Plus + Redis
- **多模块 Maven 项目**，主启动模块是 `tob-app`
- **启动后**：监听 `8081` 端口
- **依赖的外部服务**：公司阿里云 RDS 上的 MySQL 和 Redis（地址、账号、密码已写死在 `tob-app/src/main/resources/application.yml` 里，无需本地安装数据库）
- **前提条件**：你的电脑必须能连上**公司内网 / VPN**（否则连不上数据库，也拉不到依赖）

---

## 二、首次准备：装环境（只需做一次）

如果电脑上已经装过 JDK 25 和 Maven，可跳到 [第三节](#三日常启动）。

### 1. 安装 JDK 25

- 到 Oracle 官网下载 **JDK 25**（Windows x64，zip 包，不要选 installer）
- 解压到一个**纯英文、无空格**的路径，例如 `D:\jdk-25`，确保存在 `D:\jdk-25\bin\java.exe`
- 设置环境变量 `JAVA_HOME` 指向你刚解压的目录（例如 `D:\jdk-25`）

> 验证：新开一个终端，执行 `java -version`，能看到 `25.0.x` 即成功。

### 2. 安装 Maven 3.9.x

- 到 https://maven.apache.org/download.cgi 下载 **apache-maven-3.9.x-bin.zip**
- 解压到一个**纯英文、无空格**的路径，例如 `D:\apache-maven-3.9.16`
- 设置环境变量：
  - `MAVEN_HOME` 指向你解压的目录（例如 `D:\apache-maven-3.9.16`）
  - 在用户 `Path` 末尾追加 `;%MAVEN_HOME%\bin`

> 验证：新开终端，执行 `mvn -v`，能看到 `Apache Maven 3.9.x` 且 `Java version: 25.0.x` 即成功。

### 3. 配置 Maven 私服（指向公司 Nexus）

把下面的内容保存为 `C:\Users\<你的用户名>\.m2\settings.xml`（这是 Maven 的用户级配置，Maven 启动时自动读取）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 http://maven.apache.org/xsd/settings-1.0.0.xsd">
  <!-- 本地仓库：默认在用户目录下，如需改到其它盘，把下面这行的值改成你的路径 -->
  <localRepository>${user.home}/.m2/repository</localRepository>

  <servers>
    <server><id>java-proxy</id><username>admin</username><password>admin</password></server>
    <server><id>nexus-releases</id><username>admin</username><password>admin</password></server>
    <server><id>nexus-snapshots</id><username>admin</username><password>admin</password></server>
  </servers>

  <mirrors>
    <mirror>
      <id>java-proxy</id>
      <mirrorOf>*</mirrorOf>
      <name>Nexus Mirror</name>
      <url>http://inner-nexus.qdama.cn/repository/java-public/</url>
    </mirror>
    <mirror>
      <id>alimaven</id>
      <name>aliyun maven</name>
      <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
      <mirrorOf>central</mirrorOf>
    </mirror>
  </mirrors>

  <profiles>
    <profile>
      <id>nexus</id>
      <repositories>
        <repository>
          <id>nexus</id>
          <name>Nexus</name>
          <url>http://inner-nexus.qdama.cn/repository/sinoservices/</url>
          <releases><enabled>true</enabled></releases>
          <snapshots><enabled>true</enabled></snapshots>
        </repository>
      </repositories>
      <pluginRepositories>
        <pluginRepository>
          <id>nexus</id>
          <name>Nexus</name>
          <url>http://inner-nexus.qdama.cn/repository/sinoservices/</url>
          <releases><enabled>true</enabled></releases>
          <snapshots><enabled>true</enabled></snapshots>
        </pluginRepository>
      </pluginRepositories>
    </profile>
  </profiles>

  <activeProfiles>
    <activeProfile>nexus</activeProfile>
  </activeProfiles>
</settings>
```

关键点：
- `<localRepository>` 是本地仓库位置。默认 `${user.home}/.m2/repository`（即 `C:\Users\<你>\.m2\repository`），想放别的盘改成自己的路径即可（**路径不要有中文和空格**）
- 私服镜像 `inner-nexus.qdama.cn` 是公司内网地址，**必须连内网**

> 想确认 Maven 实际用了哪个本地仓库和 settings，执行 `mvn help:effective-settings`，看 `localRepository` 一行。

## 三、日常启动

每次启动后端，做这三步：

1. **连上公司内网 / VPN**
2. 打开终端，进入后端根目录（下面 `<项目根目录>` 替换为你本机 checkout 的项目路径）：
   ```bash
   cd <项目根目录>/tob-backend
   ```
   例如项目在 `D:\code\qdm-tob`，就是 `cd D:/code/qdm-tob/tob-backend`
3. 启动：
   ```bash
   mvn -pl tob-app spring-boot:run -DskipTests
   ```

> 首次启动会从私服下载大量依赖，可能要等十几分钟，属正常现象。后续启动会快很多。

看到下面这行就代表启动成功：

```
INFO  cn.qdm.tob.TobApplication - Started TobApplication in x.xxx seconds
```

启动后可以访问：

| 地址 | 说明 |
|---|---|
| http://localhost:8081/scalar | 接口文档 + 在线调试页（推荐） |
| http://localhost:8081/v3/api-docs | OpenAPI 原始 JSON |
| http://localhost:8081/ | 根路径（返回 403 是正常的，被鉴权拦截） |

停止服务：在运行 `mvn` 的终端窗口按 `Ctrl + C`。

## 四、踩坑排错（重要）

### 坑 1：启动报 `tob-backend:pom:${revision}` 解析失败

**现象**：启动时报类似下面的错：

```
Could not resolve cn.qdm:tob-backend:pom:${revision}
status code: 400, Invalid repository path
```

**原因**：项目多模块用 `${revision}` 占位符但没装 flatten 插件。每次执行过 `mvn install` 之后，本地仓库里的 pom 文件会带上 `${revision}` 字面量，导致兄弟模块解析失败。

**解决**（按顺序执行，30 秒搞定）：

1. 把本地仓库里 `cn\qdm\` 下**所有 `.pom` 文件**中的 `${revision}` 替换成 `1.0.0`
2. 删除失败产生的伪目录 `cn\qdm\tob-backend\${revision}\`

用 PowerShell 一行搞定（自动定位你的本地仓库，无需手填路径）：

```powershell
$repo = (mvn help:evaluate -Dexpression=settings.localRepository -q -DforceStdOut 2>$null)
if (-not $repo) { $repo = "$env:USERPROFILE\.m2\repository" }
Get-ChildItem -Path "$repo\cn\qdm" -Recurse -Filter *.pom -ErrorAction SilentlyContinue | ForEach-Object {
    (Get-Content $_.FullName) -replace '\$\{revision\}', '1.0.0' | Set-Content $_.FullName
}
Remove-Item -Recurse -Force "$repo\cn\qdm\tob-backend\${revision}" -ErrorAction SilentlyContinue
Write-Output "已修复本地仓库 pom：$repo\cn\qdm"
```

> 把上面整段保存为 `fix-revision.ps1`，遇到坑 1 时双击或 `powershell -ExecutionPolicy Bypass -File fix-revision.ps1` 执行即可。

3. 重新执行第三节的启动命令

### 坑 2：报 `Unable to find a suitable main class`

**原因**：启动命令加错了参数，把 `spring-boot:run` 跑到了非 app 模块上。

**解决**：严格按第三节的命令——`mvn -pl tob-app spring-boot:run -DskipTests`，**不要**加 `-am`，也**不要**在 `tob-app` 子目录里单独跑（必须从 `tob-backend` 根目录跑）。

### 坑 3：依赖下载失败 / 连不上私服

**原因**：没连公司内网，或私服 `inner-nexus.qdama.cn` 解析不到。

**解决**：
- 确认已连公司内网 / VPN
- 终端 `ping inner-nexus.qdama.cn` 看能否解析
- 可在启动命令后加 `-U` 强制刷新：`mvn -pl tob-app spring-boot:run -DskipTests -U`

### 坑 4：端口 8081 被占用

**现象**：`Port 8081 was already in use`

**解决**：找出占用进程并结束，或改 `tob-app/src/main/resources/application.yml` 里的 `server.port`。

### 坑 5：`java: warning: Options -source 25 ...` 或编译报版本不对

**原因**：本机 JDK 不是 25。

**解决**：`java -version` 确认是 25.0.x；如果 `JAVA_HOME` 指向了旧版本，改回 JDK 25 的路径后重开终端。

## 五、给测试人员的快速验证

启动成功后，用浏览器或 Postman 验证：

1. 打开 http://localhost:8081/scalar —— 能看到接口列表即说明服务正常
2. 大部分业务接口需要先登录拿 token（具体看接口文档里的鉴权说明）
3. `GET /` 返回 403 是**正常**的（说明鉴权过滤器生效了，不是 bug）

## 六、附：首次编译整个项目（可选）

如果想整体编译一遍（不启动），在 `tob-backend` 根目录执行：

```bash
mvn clean install -DskipTests
```

> ⚠️ 注意：执行完 `mvn install` 后，本地仓库的 pom 会带上 `${revision}`，下次启动前需按 [坑 1](#坑-1启动报-tob-backendpomrevision-解析失败) 处理一次。

## 七、环境自检清单

照此核对，全部满足即可启动：

- [ ] 已装 JDK 25，`java -version` 显示 `25.0.x`
- [ ] `JAVA_HOME` 指向 JDK 25 目录
- [ ] 已装 Maven 3.9.x，`mvn -v` 正常且识别到 Java 25
- [ ] `C:\Users\<你>\.m2\settings.xml` 已配置（含私服镜像）
- [ ] 已连公司内网 / VPN
- [ ] 在 `tob-backend` 目录执行 `mvn -pl tob-app spring-boot:run -DskipTests`
