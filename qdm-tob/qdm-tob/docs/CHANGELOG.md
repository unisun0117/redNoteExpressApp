# CHANGELOG

## 2026-07-14

- 前端-小程序：订单提交后自动调起微信支付，结果页按支付状态展示不同 UI（success/cancel/fail）

## 2026-07-14

- **后端** - `支付/退款`：支付金额单位由分改为元，下单接口 `amount`(Integer/分) → `amountYuan`(BigDecimal/元)，退款接口 `refundAmount`(Long/分) → `refundAmountYuan`(BigDecimal/元)。⚠️ 接口契约变更：前端需同步修改字段名和传值（元而非分）
- **前端-小程序** - `支付页`：下单接口字段 `amount` → `amountYuan`，传入金额由分改为元（可测点：输入 0.01 元可正常调起支付，不再出现金额异常放大 100 倍）

## 2026-07-13

- **后端** - `单据号生成`：新增 `BillNoGenerator`（基于 Redis 原子自增 + 按天自动归零），提供 `nextOrderNo()`（格式 `ORD+日期+4位序号`）、`nextTransactionNo()`（格式 `FIN+日期+6位序号`），零 DB 配置，注入即用（可测试：每日首次调用序号从 0001 开始，跨天自动重置）
- **前端-小程序** - `购物车`：「去结算」按钮接通结算预览页，勾选商品后点击跳转至确认订单页（可测点：无勾选时提示"请选择商品"，有勾选时正常跳转）
- **前端-小程序** - `结算页`：收货地址改为接口调用，与首页头部地址选择逻辑一致，自动取首个已审核通过的地址（可测点：无地址时提示选择，有地址时展示联系人/电话/地址；点击地址卡片可跳转地址列表）
- **前端-小程序** - `地址列表`：已通过地址点击行为改为"选中并返回"，选中后自动回退上一页并刷新地址显示（可测点：首页/结算页进入地址列表，点击已通过地址直接返回并更新顶部地址）
- **后端** - `订单提交`：提交订单接口同步调用微信支付下单（仅微信支付时触发），支付失败不影响订单创建，静默降级（可测点：微信支付成功时返回 payParams 字段，失败时 payParams 为 null，订单仍正常创建）⚠️ 接口契约变更：OrderSubmitViewVO 新增 payParams 可选字段
