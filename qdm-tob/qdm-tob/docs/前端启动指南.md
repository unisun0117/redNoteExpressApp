# 前端启动指南（tob-frontend）

> 适用人群：产品、开发、测试——任何需要在本机把前端跑起来的人。
> 与 [后端启动指南](./后端启动指南.md) 配套阅读。文中的路径仅为示例，可按你自己的习惯调整。

---

## 一、它是什么

`tob-frontend` 下有三套独立的前端应用，共用 **pnpm** 作为包管理器：

| 目录 | 应用 | 技术栈 | 本地端口 | 包管理器 |
|---|---|---|---|---|
| `tob-frontend-web` | 后台管理端（PC 网页） | Vue 3 + Vite 6 + TS | 8080 | pnpm |
| `tob-frontend-h5` | 企业微信 H5 页面 | Vue 3 + Vite 6 + TS | 5173 | pnpm |
| `tob-frontend-mobile` | 微信小程序 | uni-app + Vue 3 + Vite 5 | — | pnpm |

⚠️ **重要**：三套前端的接口地址默认都指向**公司远程测试环境**，不是你本机的后端（8081）。也就是说，前端可以**单独跑**做界面调试，不需要先启动本地后端。如果想让你跑的前端连你本机的后端，看 [第六节](#六让前端连你本机的后端可选）。

---

## 二、首次准备：装 Node 和 pnpm（只需做一次）

### 前置要求

- **Node.js ≥ 18**（推荐 **20 LTS**）。Vite 6 和 pnpm 都不支持低版本 Node。
- **pnpm** 包管理器。

### 1. 安装 Node 20

如果本机已有 Node 且版本 ≥ 18，可跳过。否则推荐用 **nvm**（Windows 版）管理多版本：

```bash
nvm install 20
nvm use 20
```

> 没装 nvm 的，到 https://github.com/coreybutler/nvm-windows/releases 下载安装；或直接到 https://nodejs.org 下载 Node 20 LTS 安装包。

> 验证：`node -v` 看到 `v20.x.x` 即成功。

### 2. 启用 pnpm

```bash
pnpm -v
```

如果提示找不到 pnpm，用 Node 自带的 corepack 启用：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

> 验证：`pnpm -v` 能输出版本号即成功。

### 3. 关于网络

- 只跑前端做界面调试：**不需要**内网（但接口请求会失败，页面能打开）
- 要让前端真正调通接口：需要**公司内网/VPN**（因为默认连的是公司测试环境后端）

## 三、运行后台管理端（tob-frontend-web）

```bash
# <项目根目录> 替换为你本机 checkout 的项目路径
cd <项目根目录>/tob-frontend/tob-frontend-web

# 首次：安装依赖（几分钟，下载量大）
pnpm install

# 启动开发服务
pnpm dev
```

启动后浏览器打开 **http://localhost:8080**。

**接口地址**（见 `.env.development`）：
```
VITE_API_BASE_URL=https://b2b-buy-qa.qdama.cn/qdm-b2b/api
```
默认连 QA 环境，需要内网。

---

## 四、运行企业微信 H5（tob-frontend-h5）

```bash
cd <项目根目录>/tob-frontend/tob-frontend-h5

# 首次安装依赖（如果 node_modules 已存在可跳过）
pnpm install

# 启动
pnpm dev
```

启动后浏览器打开 **http://localhost:5173**。

**接口地址**（见 `.env.development` + `vite.config.ts`）：
- 前端请求 `/api/...`
- Vite 把 `/api` 代理到 `https://ebwmstest.qdama.cn:19011`（测试环境后端）
- 需要内网

> 这套是面向企业微信的 H5，部分功能（如企微登录）需要在企业微信客户端内打开才完整生效。

## 五、运行微信小程序（tob-frontend-mobile）

这套是 **uni-app 微信小程序**，流程和前两套不同：需要先编译产物，再用**微信开发者工具**打开。

### 步骤

1. 安装依赖（如果 `node_modules` 已存在可跳过）：
   ```bash
   cd <项目根目录>/tob-frontend/tob-frontend-mobile
   pnpm install
   ```

2. 编译微信小程序产物（开发模式，会持续监听文件变化）：
   ```bash
   pnpm dev:mp-weixin
   ```
   产物输出到 `dist/dev/mp-weixin/`。

3. 打开**微信开发者工具**，导入项目：
   - 项目目录选 `tob-frontend-mobile/dist/dev/mp-weixin`
   - AppID 用 `project.config.json` 里配置的，或选"测试号"
   - 即可在模拟器/真机预览

> 打包给测试/上线用：
> - 测试包：`pnpm build:mp-weixin:test`
> - 生产包：`pnpm build:mp-weixin:prod`
> 产物在 `dist/build/mp-weixin/`。

> 微信开发者工具下载：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

## 六、让前端连你本机的后端（可选）

默认前端连的是远程测试环境后端。如果你改了后端代码，想让前端打到本机 `localhost:8081`（需先按 [后端启动指南](./后端启动指南.md) 启动后端）：

### 后台管理端（web）
编辑 `tob-frontend-web/.env.development`，把接口地址改成本地：
```
VITE_API_BASE_URL=http://localhost:8081
```
（注意路径前缀：本机后端的接口不带 `/qdm-b2b/api`，按实际接口路径调整。）

### H5
编辑 `tob-frontend-h5/vite.config.ts` 里的 proxy target：
```ts
proxy: {
  '/api': {
    target: 'http://localhost:8081',
    changeOrigin: true,
    secure: false,
    rewrite: (path) => path.replace(/^\/api/, ''),
  },
},
```

### 小程序
小程序连本机后端较麻烦（小程序不允许直接请求 `localhost`，需在微信开发者工具里勾选"不校验合法域名"，并改 `.env` 里的接口地址）。一般**不建议**，直接用测试环境后端即可。

## 七、常用命令速查

| 操作 | web | h5 | mobile |
|---|---|---|---|
| 装依赖 | `pnpm install` | `pnpm install` | `pnpm install` |
| 本地开发 | `pnpm dev` | `pnpm dev` | `pnpm dev:mp-weixin` |
| 打包-测试 | `pnpm qa` | `pnpm build:test` | `pnpm build:mp-weixin:test` |
| 打包-生产 | `pnpm pro` | `pnpm build:prod` | `pnpm build:mp-weixin:prod` |
| 预览构建产物 | `pnpm preview` | `pnpm preview` | 用微信开发者工具 |
| 跑单元测试 | `pnpm test` | `pnpm test` | `pnpm test` |
| 代码风格检查 | `pnpm lint` | — | — |

> 装依赖和运行命令都要在**对应子项目目录**里执行（`tob-frontend-web` / `tob-frontend-h5` / `tob-frontend-mobile`），不要在 `tob-frontend` 外层跑。

## 八、常见问题

### Q1：`pnpm install` 报错 / 卡住
- 确认 Node 版本 ≥ 18（`node -v`），推荐 Node 20
- 确认 pnpm 可用（`pnpm -v`）
- 网络问题：pnpm 默认走 npm 源，可换淘宝镜像加速：
  ```bash
  pnpm config set registry https://registry.npmmirror.com
  ```

### Q2：`pnpm dev` 启动后页面打开但接口报错 / 跨域
- 大概率是没连公司内网，前端连不上远程测试环境后端
- 或后端服务没起 / 接口地址配错

### Q3：web 端口 8080 被占用
改 `tob-frontend-web/vite.config.ts` 里 `server.port`，或关掉占用 8080 的程序。

### Q4：小程序编译报错 `@dcloudio/...` 找不到
- 确认在 `tob-frontend-mobile` 目录下执行了 `pnpm install`
- uni-app 依赖较多，首次安装要等一会

### Q5：Node 版本切来切去搞乱了
用 nvm 管理：
```bash
nvm list          # 看已装版本
nvm use 20        # 切到 20
```

- [ ] 已启用 pnpm，`pnpm -v` 正常
- [ ] 在对应子项目目录（`tob-frontend-web` / `tob-frontend-h5` / `tob-frontend-mobile`）下执行了 `pnpm install`
- [ ] 调通接口需连公司内网 / VPN
