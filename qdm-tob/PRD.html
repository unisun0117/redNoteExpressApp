📋 项目简介

QDM-TOB 是一个面向企业客户的 B2B 电商管理平台，覆盖商品管理、客户管理、订单处理、运营管理等核心业务场景。

🎯 核心价值

为企业提供一站式 B2B 交易管理能力：运营人员在 PC 后台管理商品、订单、客户；客户通过小程序浏览下单；业务员通过企业微信 H5 进行客户拜访与订单跟进。

🔧 后端规范

五层架构

层目录职责禁止
Controllerapi/admin/ api/mall/接收请求、调 Service、返回 ResponseResult业务逻辑、直接 Mapper
Serviceservice/业务逻辑、事务、调 Mapper实现 API 接口、处理 HTTP
Mappermapper/数据库 CRUD业务逻辑
VOvo/入参/出参 + 校验注解业务逻辑
Domaindomain/实体字段映射 @TableName作为 Controller 出参

API 规范速查

关键约束

✅ 统一返回 ResponseResult<T>   ✅ 资源 ID 用 @RequestParam   ✅ Entity ↔ VO 用 MapStruct   ✅ 异常 → TobServiceException   ✅ Excel 用 @Exportable / @Importable   ❌ 禁止 @PathVariable   ❌ 禁止手写 Entity↔VO 转换   ❌ 禁止 Controller 使用 MultipartFile

🗄️ 数据库规范

规范项要求
表前缀sys_ cst_ ord_ prd_ 按模块区分
引擎InnoDB
字符集utf8mb4
表名前缀隔离不同前缀 禁止 JOIN，通过应用层组合或 Spring Event
标准字段created_at updated_at created_by updated_by status
枚举类型使用 MySQL ENUM 类型
字段注释所有字段必须加 COMMENT
迁移工具Flyway，SQL 文件放 db/migration/

📝 PRD 模板使用指南 通用

本文件既是 QDM-TOB 的 PRD，也是新项目的模板 复制本文件到新项目，替换内容即可。文件命名：PRD.html，放在项目根目录或 product/ 下。

新项目使用步骤

步骤操作
1复制 PRD.html 到新项目 product/ 目录
2修改「项目简介」：项目名、简介、核心价值
3修改「关键指标」：技术栈、版本等
4修改「功能模块」表：按实际业务填写
5修改「技术栈」：去掉不需要的，加上新的
6修改「目录结构」：调整项目实际目录
7修改「设计系统」：换成项目的设计规范
8修改「质量门禁」：根据项目调整检查项
