流程说明

必须环节（加粗边框） - 契约生成、TDD 开发、单元测试、归档是核心质量保障，不可跳过
推荐环节 - 初始规范生成（首次）、需求孵化、计划创建、代码审查提升质量
质量卡点 - 架构师/开发审查和验证检查确保质量

核心命令序列

# 0. 生成项目初始规范（首次使用）
/opsx:init
# 基于现有项目代码生成初始规范

# 1. 需求孵化（推荐）
/brainstorming

# 2. 契约生成 ✓ 必须
/opsx:propose [feature-name]
# 架构师/开发审查 proposal.md/design.md/tasks.md

# 3. 创建 TDD 计划（推荐）
/superpowers:write-plan

# 4. TDD 开发 ✓ 必须
/test-driven-development
# 或使用子代理驱动
/subagent-driven-development

# 5. 单元测试 ✓ 必须
# 编写并运行单元测试，确保测试覆盖率
npm test  # 或其他测试命令

# 6. 代码审查（推荐）
/review
/codex
/cso

# 7. 归档 ✓ 必须
/opsx:archive

四大必须环节

  1. 契约生成 (OpenSpec /opsx:propose) - 将发散需求强制收敛为机器可读契约，架构师必须签字确认，消除歧义
  2. TDD 开发 (Superpowers /test-driven-development) - 基于契约任务执行测试驱动开发，确保代码质量和测试覆盖
  3. 单元测试 - 编写并运行单元测试，确保测试覆盖率达标，所有测试必须通过
  4. 归档 (OpenSpec /opsx:archive) - 合并增量规范到主规范，归档变更记录，形成知识沉淀
