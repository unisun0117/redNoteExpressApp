# 《数据库从0到1》专栏大纲

> 面向前端开发者，用金橘记账项目真实代码学数据库

---

## 10 篇标题

### 第 1 篇：前端为什么需要数据库？— 从 localStorage 到 SQLite

- **前端类比**：`localStorage.setItem()` → 数据库 INSERT
- **核心观点**：你已经在用数据库了，只是没意识到——localStorage 就是一个单表的 NoSQL 数据库
- **会用到的真实代码**：金橘记账的 SQLite 文件 `finance.db`，跟你的 `localStorage` 对比
- **阿珊故事**：我第一个项目真用 localStorage 存用户数据，浏览器一清缓存全没了...

### 第 2 篇：数据库怎么存你的数据？— 表、行、列的前端视角

- **前端类比**：数据库表 = `Array<Object>`，一行 = 一个 Object，一列 = 一个 key
- **核心观点**：你每天都在操作"表"——`users.map(u => u.email)` 就是一次 SELECT
- **会用到的真实代码**：`User.__tablename__ = "users"`，对比 `const users: User[] = []`
- **阿珊故事**：我第一次听到"表"以为跟 Excel 一样，结果发现比 Excel 严格多了

### 第 3 篇：装一个数据库 — PostgreSQL 环境搭建 + 第一个 SQL 查询

- **前端类比**：`npm install react` → `brew install postgresql`
- **核心观点**：装数据库比装 node_modules 还快，真正花时间的是理解它怎么跑
- **会用到的真实代码**：从 SQLite `sqlite:///./finance.db` 到 PostgreSQL 连接字符串
- **动手**：安装 PostgreSQL → 创建数据库 → 用 DBeaver 可视化 → 执行 `SELECT 1`

### 第 4 篇：CRUD 的前端翻译 — INSERT/SELECT/UPDATE/DELETE 就是你每天写的代码

- **前端类比**：`array.push()` = INSERT，`array.filter()` = SELECT，`obj.xxx = yyy` = UPDATE，`array.splice()` = DELETE
- **核心观点**：CRUD 没有新东西，你写前端的时候每天都在做，只是换了个语法
- **会用到的真实代码**：`routers/transactions.py` 里的 `db.add(tx)` / `db.query().filter()` / `db.delete(tx)`
- **动手**：在金橘记账数据库里手写 SQL 增删改查

### 第 5 篇：三张表教会你数据库设计 — 主键、外键、关联关系

- **前端类比**：PRIMARY KEY = React 的 `key={item.id}`，FOREIGN KEY = `{ userId: "abc" }` 引用另一个数组
- **核心观点**：数据库设计不是玄学，就三件事——找实体、定字段、建关联
- **会用到的真实代码**：User (主表) → Transaction (外键 user_id) → Budget (外键 user_id + month)
- **阿珊故事**：我第一版把所有数据塞一张表里，查个"用户本月花了多少钱"写了 50 行代码...

### 第 6 篇：你不是一个人在写 SQL — ORM 怎么帮你自动生成查询

- **前端类比**：ORM = JSX → `document.createElement()`，你写 Python 类，ORM 生成 SQL
- **核心观点**：ORM 不是魔法，它只是把你写的 `db.query(User).filter(...)` 翻译成了 `SELECT * FROM users WHERE...`
- **会用到的真实代码**：`models/` 下三张表的 Mapped 定义 → SQLAlchemy 自动生成的 SQL → 对比手写 SQL
- **动手**：打开 SQLAlchemy echo=True，看每次请求背后生成了什么 SQL

### 第 7 篇：数据库的 10 个坑，我都帮你踩过了

- **前端类比**：`useEffect` 死循环 → 数据库死锁，`setState` 异步 → 事务未提交
- **核心观点**：数据库的坑有个规律——80% 跟"你以为存进去了但其实没有"有关
- **真实坑点**：金额用 float → 精度丢失 / SQLite like 跟 PostgreSQL 不兼容 / commit 忘记调用 / 外键约束报错 / 连接没关导致泄漏 / bcrypt hash 跟数据库字段长度不匹配
- **阿珊故事**：部署上去第二天用户跟我说"钱少了 1 分"，查了 3 小时发现是 float 精度...

### 第 8 篇：你的查询为什么这么慢？— 索引与查询优化入门

- **前端类比**：`array.find()` 遍历 10000 条 vs `Map.get()` O(1) 查找 — 索引就是数据库的 Map
- **核心观点**：加一个索引 = 查询速度快 1000 倍。但你不需要一上来就加——等慢的时候再加
- **会用到的真实代码**：金橘记账的 `filter(tx_date.like(...))` → 加索引前后的查询计划对比
- **动手**：EXPLAIN 分析你项目的查询 → 找到最慢的那个 → 加索引 → 再看 EXPLAIN

### 第 9 篇：面试官会怎么问你数据库？— 10 道高频题 + 答题思路

- **核心观点**：面试官不在乎你背了多少概念，在乎你"有没有真正用过"
- **高频题**：SQL vs NoSQL 怎么选 / 索引原理（B+树）/ 事务 ACID / 数据库三大范式 / 慢查询怎么排查 / ORM vs 手写 SQL / 连接池原理 / 分库分表什么时候做 / SQL 注入怎么防
- **答题公式**：概念一句话 + 我的项目里怎么用的 + 遇到过什么问题

### 第 10 篇：数据库学了能干嘛？— 后端/数据分析/爬虫三条路

- **核心观点**：数据库不是终点，是基础设施。你学会数据库之后，三条路全通了
- **路线图**：全栈开发（FastAPI + PostgreSQL 做后端）→ 数据分析（Pandas + SQL 做报表）→ 爬虫/工具（数据采集 + 存储）
- **推荐资源**：PG 官方文档怎么看、DBeaver 技巧、Next 学什么
