# 金橘记账 - 整体实施方案

> 日期：2026-06-29
> 状态：开发中
> 原型：已审批通过

## 一、项目定位

温暖治愈的个人记账 Web 应用。暖色调 + 极简风，让理财不再冷冰冰。

## 二、技术栈

| 层 | 技术 |
|----|------|
| 前端 | React 19 + TypeScript + Vite + antd-mobile |
| 后端 | FastAPI + SQLAlchemy + SQLite |
| 认证 | JWT (email + password) |
| 部署 | Vercel (前端) + PythonAnywhere (后端) |

## 三、功能模块

| 模块 | 状态 | 依赖 |
|------|------|------|
| 用户认证 | 待开发 | - |
| 账单管理 | 待开发 | 用户认证 |
| 预算管理 | 待开发 | 用户认证 |
| 数据统计 | 待开发 | 账单管理 |
| 个人中心 | 待开发 | 用户认证 |

## 四、数据库表

- users (id, email, password_hash, nickname, avatar, created_at)
- transactions (id, user_id, type, amount, category, note, date, created_at)
- budgets (id, user_id, month, category, amount, created_at)

## 五、实施顺序

1. 后端基础 + 用户认证
2. 账单 CRUD
3. 预算管理
4. 数据统计
5. 个人中心
6. 前后端联调
7. 部署上线
