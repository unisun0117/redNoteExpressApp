# 阿珊-项目工厂

## Overview

输入行业 + 平台，产出项目源码 + 15 个 Lab 课程文档。

**核心机制：UI 先行，一次审批，后续全自动。**

## 为什么这样设计

```
旧流程（5 次审批）：
  头脑风暴 → [审] → 方案 → [审] → 写代码 → [审] → 大纲 → [审] → 课程 → [审]
  问题：你要一直盯着，累。

新流程（1 次审批 + 1 次抽查）：
  UI 先行（原型+设计令牌+API契约）→ 【你审一次，定稿】
      ↓
  方案 + 代码 + 课程 → 自动跑完 → 【抽查课程质量】
  优势：原型即施工图纸，前后端照着做，不需要中间确认。
```

## 第 1 步：UI 先行 — 出施工图纸（调用 ashan-ui-first-design）

**这是唯一的审批门，也是最重要的。**

### 做什么

1. **生成 DESIGN.md** — 颜色、字体、圆角、间距、动效风格
2. **画 HTML 原型** — 每个页面的静态原型（5-8 个页面）
3. **提取 API 契约** — 从原型反推每个页面需要什么接口

### 产出

```
项目目录/
├── DESIGN.md              ← 视觉规范
├── API_CONTRACT.md        ← 前后端接口契约
└── prototypes/
    ├── index.html         ← 原型导航
    ├── page-login.html
    ├── page-dashboard.html
    ├── page-list.html
    ├── page-detail.html
    ├── page-form.html
    └── page-empty-error.html
```

### 审批（这里盯一次，后面就不用盯了）

```
📋 打开 prototypes/index.html，每个页面看一遍：

□ 色调和风格是你想要的吗？
□ 每个页面的布局合理吗？
□ 页面之间的跳转逻辑对吗？
□ 有没有缺页面？
□ API 契约覆盖了所有页面的数据需求吗？

→ 确认后，原型 = 施工图纸，不再改动。
```

## 第 2 步：自动执行 — 方案 + 代码 + 课程

**以下步骤审批通过后自动串行执行，不需要中间停下来。**

### 2.1 OpenSpec 方案（基于原型自动生成）

根据 API_CONTRACT.md 和 DESIGN.md，自动生成：
- `proposal.md` — 项目定位
- `design.md` — 技术架构（后端路由、数据库模型、前端组件树）
- `tasks.md` — 15-20 个实施任务

### 2.2 写项目代码（前后端并行）

根据原型和 API 契约，前后端可以并行开发：
- 前端：照着 HTML 原型写 React 组件
- 后端：照着 API_CONTRACT.md 写 FastAPI 接口

### 2.3 启动验证

代码写完后，前后端启动，验证：
- 前端页面和原型一致
- 后端接口符合 API 契约
- 前后端联调通过

## 第 3 步：课程抽查

课程全部生成后，随机抽 2-3 个 Lab 审一下：

```
📋 抽查清单：
□ 阿珊风格够不够？（故事 + 金句 + 署名）
□ 代码示例和项目实际代码一致吗？
□ 新手跟着做能跑通吗？
```

## 审批对比

| | 旧流程 | 新流程（UI先行） |
|------|--------|-----------------|
| 审批次数 | 5 次 | **1 次（原型）+ 1 次抽查** |
| 关键决策点 | 分散在各步 | **集中在原型阶段** |
| 返工风险 | 高（后面发现方向不对） | 低（原型不对当场改） |
| 你的时间 | 一直盯 | 审原型 10 分钟，后面不管 |
