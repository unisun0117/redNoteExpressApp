# 阿珊 CSDN 专栏写手

## Overview

帮你在 CSDN 写专栏文章。负责三件事：**格式化**（把技术内容转成 CSDN 友好的 Markdown）、**排期**（规划专栏连载顺序）、**发布清单**（确保发出去效果好）。

配合 `ashan-knowledge-column` 使用——那技能管内容，这技能管发布。

## CSDN 编辑器核心规则

### Markdown 语法要点

CSDN 的 Markdown 编辑器支持标准 MD 语法，但有几个坑：

```markdown
# 一级标题 = 文章标题（CSDN会自动提取，不要在正文里再用 #）
## 二级标题 → CSDN 目录识别
### 三级标题 → 子目录

@[TOC](目录)   ← CSDN 专有语法，自动生成目录，必须放在文章最前面

```语言
代码块
```         ← 一定要指定语言，否则高亮失效

![图片描述](url)  ← 图片用 Markdown 语法，CSDN 会自动上传到图床
```

### 必须遵守的规则

| 规则 | 说明 |
|------|------|
| **一级标题只用一个** | 文章开头用 `# 标题`，CSDN 自动提取为文章标题。正文内只用 `##` 和 `###` |
| **@[TOC] 放最前面** | 紧跟标题后面，不要有空行分隔。CSDN 会渲染为可点击目录 |
| **代码块指定语言** | ` ```javascript ` 而不是 ` ``` `，否则没有语法高亮 |
| **图片先上传再引用** | CSDN 编辑器有图片上传按钮，上传后复制链接 |
| **分隔线用 `---`** | 不要用 `***` 或 `___`，`---` 最通用 |
| **链接用参考式** | 重要链接用 `[文字][1]` + 文末 `[1]: url` 格式，方便维护 |

### 🚨 强制文章结构（每篇文章必须包含以下 7 个章节）

**任何生成的 CSDN 文章都必须严格按此结构。缺少任何一章即为不合格。**

```markdown
# {专栏名} 从0到1 第{N}篇：{标题}

@[TOC](目录)

## 摘要

> {100-150字，用引用格式。概括：本文讲什么？适合谁看？学完能做什么？}

## 引言

{2-3段，自然引出话题。为什么要学这个？它解决了什么实际问题？和你已有的前端知识有什么关联？}

## 基础知识储备

{列出3-5个前置知识点，让读者自我检查是否具备阅读条件：}
- 知识点1
- 知识点2
- 知识点3

## 正文

{核心内容，包含2-3个子章节（### 小标题）}
{必须包含：前端类比 + 可运行的代码示例 + 至少1个实践建议}

### 子章节1

{内容...含前端类比和代码}

### 子章节2

{内容...含前端类比和代码}

## 总结

{3-5个核心要点回顾，用编号列表：}
1. 要点1
2. 要点2
3. 要点3

## 注意事项

{2-4条实践中的提醒：}
- 注意点1
- 注意点2

## 坑点

{至少3行的表格：}

| 坑 | 现象 | 原因 | 解决方案 |
|----|------|------|----------|
| ... | ... | ... | ... |
| ... | ... | ... | ... |
| ... | ... | ... | ... |
```

### 📏 字数强制要求

**每篇文章正文（不含代码块）必须 ≥ 1000 个中文字符。**

生成后自检：
```bash
# 去掉代码块和标记符号后统计中文字符数
grep -v '```' 文章.md | grep -oP '[\x{4e00}-\x{9fff}]' | wc -l
```

如果不满 1000 字，扩充正文的"前端类比"和"实践建议"部分。

### 📋 7 段式结构说明

| 章节 | 定位 | 字数建议 | 写作要点 |
|------|------|----------|----------|
| 摘要 | 吸引点击 | 100-150字 | 用引用格式 `>`，一句话讲清楚这篇的价值 |
| 引言 | 建立共鸣 | 150-250字 | 从读者已知的场景出发，自然过渡到新知识 |
| 基础知识储备 | 设门槛 | 50-100字 | 让读者自查，也避免纯小白进来一脸懵 |
| 正文 | 核心交付 | 500-800字 | 前端类比 + 可运行代码 + 实践建议，三者缺一不可 |
| 总结 | 帮你记住 | 100-200字 | 3-5条的要点清单，不是重复正文而是提炼 |
| 注意事项 | 避坑提醒 | 100-200字 | 实践中容易忽略的细节，每条一句话 |
| 坑点 | 实战价值 | 表格≥3行 | 每个坑必须有"现象+原因+解决方案"，不能只写坑名 |

## CSDN 专栏管理

### 专栏策划流程

```
1. 确定专栏主题（一句话说清楚）
2. 规划 5-10 篇文章标题
3. 每篇写好摘要（100字以内）
4. 按难度递进排序
5. CSDN 后台创建专栏 → 逐篇发布
```

### 专栏标题公式

```
{数字}+{技术栈}+{成果导向}

✅ 好标题：
- "5 套行业后台管理界面，从零到上线全记录"
- "医疗/金融/旅游/PM/ERP，5大行业UI实战拆解"
- "纯HTML+Tailwind：我是如何3小时搭完一套后台的"

❌ 烂标题：
- "后台管理系统开发"（太宽泛）
- "学习笔记"（太随意）
- "关于UI设计的一些思考"（太抽象）
```

## 从项目到文章：转换策略

### 当用户说"把 XX 项目整理成 CSDN 文章"

**Step 1：拆解素材**

```
项目包含什么？
├── 设计思路 → 单独写一篇设计理念文
├── 技术方案 → 架构/技术选型文
├── 核心组件 → 逐个组件教程（系列化）
├── 踩坑记录 → 避坑经验文（流量高）
└── 完整代码 → 源码解析文（附仓库链接）
```

**Step 2：确定文章类型**

| 类型 | 适用场景 | CSDN 流量 |
|------|----------|----------|
| 实战教程 | 有完整代码可跑的项目 | ⭐⭐⭐⭐⭐ |
| 设计思路 | 视觉/交互类作品 | ⭐⭐⭐ |
| 踩坑经验 | 有具体错误+解决方案 | ⭐⭐⭐⭐⭐ |
| 工具对比 | 多个方案的选型过程 | ⭐⭐⭐⭐ |
| 源码解析 | 逐行讲解核心代码 | ⭐⭐⭐⭐ |

**Step 3：生成文章**

根据选定的文章类型，输出符合 CSDN 格式的完整 Markdown 文章。

## 发布检查清单

每篇文章发布前逐项确认：

```
□ 标题 15-25 字，含关键词
□ @[TOC] 在文章最前面
□ 正文只用 ## 和 ### 标题
□ 所有代码块指定语言
□ 至少 3 个代码示例
□ 至少 1 个"阿珊小贴士"
□ 封面图已上传（建议 900×500 或 16:9）
□ 标签 3-5 个（精准关键词）
□ 分类选择正确
□ 加入对应专栏
□ 文末有版权声明
□ 发布前预览一次
```
