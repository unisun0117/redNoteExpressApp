# RedNote Express（红薯快写）项目文档

## 一、项目概述

一个小红书风格文章生成工具。用户输入关键词 + 选风格 + 上传图片，由 DeepSeek AI 生成文章。

---

## 二、技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 前端框架 | React 19 + TypeScript 6 | Vite 8 构建 |
| UI 组件库 | antd-mobile 5 | 移动端 UI |
| 路由 | react-router-dom 6 | 前端路由 |
| 后端框架 | FastAPI (Python 3.10) | REST API |
| 数据库 | SQLite（线上）/ PostgreSQL（可选） | 当前云端用 SQLite |
| ORM | SQLAlchemy 2.0 | 数据库操作 |
| 认证 | JWT (python-jose) | Bearer Token 认证 |
| 密码哈希 | passlib + bcrypt 4.0.1 | ⚠️ 不能用 bcrypt 5.x |
| AI 服务 | DeepSeek API | OpenAI 兼容接口 |
| ASGI 桥接 | asgiref | PythonAnywhere 的 WSGI → ASGI |

## 七、常见问题 & 坑点

### ⚠️ bcrypt 版本
`passlib` 不兼容 `bcrypt 5.x`，必须用 `bcrypt==4.0.1`。本地后端要用 venv 里的 uvicorn，别用系统全局的：
```powershell
e:\AI_Workspace\redNoteExpressApp\backend\.venv\Scripts\uvicorn.exe app.main:app --reload --port 8000
```

### ⚠️ PythonAnywhere SQLite 路径
必须是 4 个斜杠 `sqlite:////home/...` 不是 3 个。数据库文件必须放在 web 进程有权限访问的位置（home 目录下）。

### ⚠️ PythonAnywhere 免费版限制
- 不能用 PostgreSQL（外部连接被屏蔽）
- 不支持原生 ASGI，需要 WSGI 桥接
- 代码路径是 `~/redNoteExpressApp/redNoteExpressApp/backend`（两层嵌套）

### ⚠️ Vercel 构建失败
`tsc -b` 会报 `verbatimModuleSyntax` 错误。把 Build Command 改成 `npx vite build` 跳过类型检查。

### ⚠️ 本地 vs 线上分支
GitHub 分支是 `master`，不是 `main`。Vercel 设置里 Production Branch 要改成 `master`。

### ⚠️ GitHub 推送
国内网络到 GitHub 不稳定。如果推送超时，可以过一会重试，或者用 PythonAnywhere 的 Bash 执行 git push。
GitHub 账号：`unisun0117`，Token：`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`

### ⚠️ 后端重启
改了 WSGI 文件后必须去 Web 页面点 **Reload** 按钮才能生效。

### ⚠️ 查看错误
后端报错看 PythonAnywhere 错误日志，前端报错看浏览器 F12 → Console。
