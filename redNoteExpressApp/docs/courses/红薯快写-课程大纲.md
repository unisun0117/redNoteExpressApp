# 《红薯快写》全栈实战课程大纲

## 课程介绍
- **适合人群：** 大学生做毕设、前端初学者入门全栈、想做 AI 项目的任何人
- **前置知识：** HTML/CSS/JS 基础即可，Python 零基础也能跟
- **你能学到：** React 19 + Vite 工程化、FastAPI 后端开发、JWT 用户认证、AI API 对接（DeepSeek）、图片上传处理、SQLAlchemy 数据库设计、移动端 UI（antd-mobile）、点数计费系统、Vercel + PythonAnywhere 部署、项目变现方法论
- **项目源码：** https://github.com/unisun0117/redNoteExpressApp
- **作者：** 阿珊，前端开发者 & AI 提效实践者

---

## Lab 1-12：全栈项目实战

### Lab 01：项目介绍与环境搭建 — Node.js + Python 双环境配置

- **📌 前言：** "这个项目的起因很简单——我接了个外包，甲方要一个小红书文案生成器。我想与其手写 10 篇文案，不如让 AI 来写。结果这个 App 不仅交付了甲方，还成了我的课程素材。跟着我做，你不仅能学会全栈开发，还能把它改成你自己的毕设。"
- **📚 基础知识储备：** 命令行基础（cd、mkdir）、什么是前端/后端、Node.js 是什么、Python venv 是什么
- **🎯 实战目标：** 安装 Node.js + Python、克隆项目代码、理解前后端目录结构
- **🛠 动手实战：** 安装 Node.js → 安装 Python 3.10 → 创建 venv → 克隆仓库 → 安装前端依赖 → 安装后端依赖 → 验证环境
- **⚠️ 常见坑点：** Python 版本不对、pip 安装超时、node-gyp 报错
- **📝 总结：** 环境配通，项目就跑通了一半。下一个 Lab 我们启动后端，看到第一个 API。

---

### Lab 02：FastAPI 后端启动 — 从零看到第一个 API

- **📌 前言：** "我第一次打开 FastAPI 的 `/docs` 页面时，真的被震撼了——自动生成的 Swagger 文档，接口可以直接在网页上测试。甲方看了一遍就确认了需求，省了我不知道多少沟通成本。"
- **📚 基础知识储备：** 什么是 API、什么是 REST、HTTP 方法（GET/POST）、JSON 格式
- **🎯 实战目标：** 理解 FastAPI 项目结构、启动后端服务、在浏览器里看到自动生成的 API 文档
- **🛠 动手实战：** 理解 `main.py` 入口 → 理解 `config.py` 配置 → 启动 uvicorn → 访问 `/docs` → 在线测试 `/api/health`
- **⚠️ 常见坑点：** 必须用 venv 里的 uvicorn 不是系统全局的、端口被占用、CORS 报错
- **📝 总结：** 第一个 API 跑通了！接下来我们把数据库也跑起来。

---

### Lab 03：SQLAlchemy 数据库 — 用户表建起来

- **📌 前言：** "建数据库这件事，我第一个项目就搞砸过——表结构乱、字段命名随便、后来想改都改不动。这次用 SQLAlchemy ORM，改字段只需要改 Python 代码，数据库自动同步。这个习惯救了我好几次。"
- **📚 基础知识储备：** 什么是数据库、什么是表/字段、SQLite vs PostgreSQL、ORM 是什么
- **🎯 实战目标：** 理解 SQLAlchemy 模型定义、创建 User 表、用命令行验证表结构
- **🛠 动手实战：** 阅读 `database.py` → 阅读 `models/user.py` → 理解 DeclarativeBase → 自动建表 → 用 DB Browser 查看表结构
- **⚠️ 常见坑点：** SQLite 路径问题（4 个斜杠）、check_same_thread 参数、字段类型映射
- **📝 总结：** 数据库跑通了。下一个 Lab 我们让用户能注册和登录。

---

### Lab 04：JWT 用户认证（上）— 注册和登录接口

- **📌 前言：** "JWT 是我踩过最深的一个坑。当时死活调不通，debug 两小时发现 bcrypt 版本装错了——5.x 和 passlib 不兼容，必须用 4.0.1。以后你碰到鉴权问题，先查依赖版本。"
- **📚 基础知识储备：** 什么是认证/授权、什么是哈希（Hash）、JWT 是什么（Token 的工作方式）
- **🎯 实战目标：** 实现邮箱注册接口、实现登录接口、理解 JWT token 的生成和验证
- **🛠 动手实战：** 阅读 `routers/auth.py` → 理解 bcrypt 密码哈希 → 理解 JWT 签发逻辑 → Postman 测试注册 → Postman 测试登录 → 拿到 token
- **⚠️ 常见坑点：** bcrypt 必须 4.0.1 不能 5.x、email 字段唯一约束、token 过期时间
- **📝 总结：** 注册登录跑通了。下一个 Lab 我们把前端接上。

---

### Lab 05：JWT 用户认证（下）— 前端登录页 + Token 管理

- **📌 前言：** "前端做登录的时候，我以为调一个 API 就完了。结果 401 错误把我整懵了——token 过期了怎么办？刷新 token 怎么搞？这一 Lab 我把整套逻辑拆开给你看。"
- **📚 基础知识储备：** React Context 是什么、localStorage 怎么用、fetch API 基础、Bearer Token 是什么
- **🎯 实战目标：** 实现前端登录页面、理解 token 存储和自动刷新机制、实现路由守卫
- **🛠 动手实战：** 阅读 `hooks/useAuth.tsx` → 阅读 `services/api.ts` 的 authFetch → 阅读 `pages/LoginPage.tsx` → 理解 PrivateRoute 守卫 → 跑通完整的登录流程
- **⚠️ 常见坑点：** localStorage token 丢失、401 后没自动刷新、refresh token 过期处理
- **📝 总结：** 登录全流程跑通了。下一个 Lab 我们开始做核心功能——AI 生成文章。

---

### Lab 06：AI 接口对接 — 调用 DeepSeek 生成文章

- **📌 前言：** "对接 AI 接口这件事，说难不难，说简单也不简单。关键不是调 API，是写 Prompt。我给 AI 的指令越清晰，它吐出来的东西越靠谱。这 Lab 我把我的 Prompt 模板全分享出来。"
- **📚 基础知识储备：** 什么是 LLM/大模型、什么是 API Key、什么是 Prompt（提示词）、OpenAI SDK 是什么
- **🎯 实战目标：** 配置 DeepSeek API Key、理解 ArticleGenerator 的 Prompt 构建、调用 AI 生成第一篇小红书文案
- **🛠 动手实战：** 获取 DeepSeek API Key → 配置环境变量 → 阅读 `services/generator.py` → 理解 Prompt 模板（风格 + 赛道 + emoji + 小标题）→ Postman 测试 `/api/generate` → 拿到第一篇 AI 生成的文章
- **⚠️ 常见坑点：** API Key 放在 `OPENAI_API_KEY` 不是专属变量、网络超时、DeepSeek 余额不足
- **📝 总结：** AI 能写文章了！下一个 Lab 我们让前端把文章漂亮地展示出来。

---

### Lab 07：前端生成页面 — 图片上传 + 参数配置

- **📌 前言：** "前端做表单的时候，最容易忽略的是用户体验。上传图片没预览？用户不知道传没传成功。生成按钮没 loading 状态？用户以为卡死了连点好几下，点数全扣光了。"
- **📚 基础知识储备：** React 受控组件、FormData、File API、useState/useEffect
- **🎯 实战目标：** 实现完整的生成页面——图片上传预览、关键词输入、风格赛道选择、一键生成
- **🛠 动手实战：** 阅读 `pages/GeneratorPage.tsx` → 理解 FormData 组装 → 理解 `components/ImageUpload.tsx` → 理解 `components/StylePanel.tsx` → 完整跑通一次生成流程
- **⚠️ 常见坑点：** 图片超过 10MB 限制、FormData 字段名和后端不一致、emoji 开关状态管理
- **📝 总结：** 前端生成流程跑通了。下一个 Lab 我们把生成结果展示优化一下。

---

### Lab 08：结果展示 + 一键复制 — 让生成结果好看又好用

- **📌 前言：** "做好了生成功能，用户说『能不能一键复制？』我一开始只加了复制按钮，结果用户又说『能不能分段复制？』产品就是这样，用起来才知道缺什么。"
- **📚 基础知识储备：** React 条件渲染、Clipboard API、Markdown 渲染
- **🎯 实战目标：** 实现结果展示组件、实现一键复制全文、实现错误提示和 loading 状态
- **🛠 动手实战：** 阅读 `components/ResultView.tsx` → 实现复制到剪贴板 → 添加错误提示样式 → 添加生成中 loading 动画
- **⚠️ 常见坑点：** Clipboard API 在非 HTTPS 下不可用、copyText 拼接格式问题
- **📝 总结：** 前端展示完美了。下一个 Lab 我们实现风格系统。

---

### Lab 09：风格系统设计 — 4 种模板 + 9 个赛道

- **📌 前言：** "做风格系统的时候我悟了：好产品不是功能多，而是选择少而精。我只放了 4 种风格、9 个赛道——每个都是我反复调 Prompt 调出来的。用户不需要 50 个选项，他们需要 4 个好用的。"
- **📚 基础知识储备：** 什么是 System Prompt、什么是枚举（Enum）、前端下拉组件
- **🎯 实战目标：** 理解风格模板的 Prompt 设计、实现风格下拉选择、理解 localStorage 持久化用户偏好
- **🛠 动手实战：** 阅读 `routers/styles.py` → 阅读 `services/generator.py` 的 STYLE_PROMPTS 和 TRACK_PROMPTS → 新增一种自定义风格模板 → 前端验证新风格效果
- **⚠️ 常见坑点：** 新增风格后后端没重启、前端缓存了旧的风格列表
- **📝 总结：** 风格系统做好了。下一个 Lab 我们加上点数计费。

---

### Lab 10：点数计费系统 — 让 App 能赚钱

- **📌 前言：** "做收费功能之前，我想了很久——怎么定价？免费给多少？最后定了『新用户送 10 次，后续 9.9 元/100 次』。这定价不是拍脑袋想的，我调研了 5 款 AI 写作工具，取中间值。做产品，定价就是定生死。"
- **📚 基础知识储备：** 什么是 Freemium 模式、数据库原子操作、HTTP 状态码（402）
- **🎯 实战目标：** 理解点数扣减逻辑、实现余额不足拦截、实现前端显示剩余次数
- **🛠 动手实战：** 阅读 `routers/generate.py` 的扣点逻辑 → 阅读 `routers/billing.py` → 理解 UserTier 枚举 → 测试点数耗尽后的 402 响应 → 前端显示剩余次数
- **⚠️ 常见坑点：** 并发请求导致重复扣点、点数扣了但生成失败没回滚
- **📝 总结：** 点数系统跑通了。下一个 Lab 我们把 App 部署上线。

---

### Lab 11：前端部署 — Vercel 一键上线

- **📌 前言：** "部署是我以前最怕的环节——像开盲盒，不知道哪里会炸。但 Vercel 真的是前端部署的天花板——GitHub 一推，自动构建，自动上线。免费额度足够个人项目用了。"
- **📚 基础知识储备：** 什么是 CI/CD、什么是静态网站托管、Vercel 是什么
- **🎯 实战目标：** 注册 Vercel、关联 GitHub 仓库、配置环境变量、部署前端
- **🛠 动手实战：** 注册 Vercel（GitHub 登录）→ 导入仓库 → 设置 Root Directory 为 `frontend` → Build Command 为 `npx vite build` → 配置 `VITE_API_URL` → 部署 → 访问线上地址
- **⚠️ 常见坑点：** `tsc -b` 构建失败必须用 `npx vite build`、Production Branch 必须是 `master`、环境变量没配前端报 CORS
- **📝 总结：** 前端上线了！下一个 Lab 把后端也部署上去。

---

### Lab 12：后端部署 — PythonAnywhere 实战

- **📌 前言：** "后端部署比前端麻烦多了——PythonAnywhere 免费版不支持原生 ASGI，得用 WSGI 桥接。数据库路径要 4 个斜杠。代码改完要手动点 Reload。这一 Lab 就是你的『后端部署避坑指南』。"
- **📚 基础知识储备：** 什么是 WSGI/ASGI、什么是 PythonAnywhere、环境变量在服务端怎么配
- **🎯 实战目标：** 注册 PythonAnywhere、上传后端代码、配置 WSGI 桥接、后端上线
- **🛠 动手实战：** 注册 PythonAnywhere → 上传代码 → 配置 WSGI 文件 → 配置环境变量 → 创建 SQLite 数据库 → 点 Reload → 访问 `/api/health` 验证
- **⚠️ 常见坑点：** SQLite 路径 4 个斜杠、改完代码必须点 Reload、免费版不能连外部 PostgreSQL
- **📝 总结：** 🎉 前后端都上线了！你的红薯快写 App 现在全世界都能访问！
