## 📌 前言

做风格系统的时候，我悟了一个道理：**好产品不是功能多，而是选择少而精。**

市面上有些 AI 写作工具，给你 50 种风格、30 个赛道——看起来强大，实际上用户选半天选不出来，最后用了默认的。我只放了 4 种风格、9 个赛道——每个都是我反复调 Prompt 调出来的。用户不需要 50 个选项，他们需要 4 个好用的。

这一 Lab，我带你看看每种风格背后的 Prompt 是怎么设计的——你甚至可以自己加一种新风格。

---

## 📚 基础知识储备

- **System Prompt 工程** — 用简洁的指令塑造 AI 的行为和语气
- **枚举管理模式** — 用字典（dict）管理有限选项，避免到处写 if-else
- **前后端风格同步** — 后端定义风格列表，前端通过 API 拉取

## 🛠 动手实战

### 步骤 1：理解 4 种风格的设计逻辑

每种风格的 Prompt 只有一句话，但每个词都经过考虑：

| 风格 | Prompt | 设计思路 |
|------|--------|----------|
| 复古风 | "使用复古怀旧的语气，加入经典词汇和年代感表达，营造温暖怀旧的氛围" | 关键词：复古、怀旧、经典、年代感。适合老字号、传统美食 |
| 简约风 | "使用简洁优雅的语气，避免冗长修饰，突出核心信息，风格清新自然" | 关键词：简洁、优雅、核心信息。适合精品店、设计师品牌 |
| 幽默风 | "使用俏皮幽默的语气，加入网络流行语和搞笑表达，让读者会心一笑" | 关键词：俏皮、幽默、流行语。适合年轻人、网红店 |
| 深度测评风 | "使用专业细致的分析语气，多维度评测，注重细节和对比，给人可信赖感" | 关键词：专业、多维度、对比。适合数码产品、护肤品 |

### 步骤 2：理解 9 个赛道的人设设计

每个赛道给 AI 一个人设身份：

```python
"美食": "你是一个美食探店博主，专注分享美食体验和餐厅推荐。"
"运动": "你是一个运动健身博主，专注分享运动体验和健康生活方式。"
"摄影": "你是一个摄影博主，专注分享摄影技巧和视觉美学。"
"萌宠": "你是一个宠物博主，专注分享萌宠日常和养宠心得。"
"家居": "你是一个家居博主，专注分享家居设计和生活美学。"
"美妆": "你是一个美妆博主，专注分享化妆技巧和护肤心得。"
"数码": "你是一个数码博主，专注分享数码产品评测和科技资讯。"
"母婴": "你是一个母婴博主，专注分享育儿经验和母婴好物。"
"餐饮": "你是一个餐饮行业博主，专注分享餐饮经营和店铺推荐。"
```

**人设的作用：** "你是一个美食博主" → AI 会更倾向于用美食领域的词汇和表达方式。同一个咖啡店，用"美食"赛道和"数码"赛道，写出来的角度完全不同。

### 步骤 3：理解风格 API `routers/styles.py`

```python
@router.get("/styles")
def get_styles():
    return {"templates": STYLE_TEMPLATES, "tracks": TRACKS}
```

**设计用意：** 这个接口不需要登录！任何人（包括没注册的用户）都能看到有哪些风格可选。前端在 StylePanel 加载时调用这个接口，拿到最新的风格列表。

### 步骤 4：实战 — 自己加一种新风格

**目标：** 增加"ins风"（Instagram 风格）

**第一步：** 在 `services/generator.py` 的 `STYLE_PROMPTS` 里加：
```python
"ins风": "使用Instagram风格的表达方式，简洁精致，多用短句和视觉描述词汇，营造高级感和生活美学。",
```

**第二步：** 在 `routers/styles.py` 的 `STYLE_TEMPLATES` 里加：
```python
{"id": "ins", "name": "ins风", "description": "Instagram-style, clean and aesthetic"},
```

**第三步：** 在前端 `components/StylePanel.tsx` 的 `DEFAULT_TEMPLATES` 里加：
```typescript
{ id: "ins风", name: "ins风" },
```

**第四步：** 重启后端，刷新前端，下拉菜单里看到"ins风"。
**第五步：** 选 ins风，生成一篇文章，看效果。

**验证：** 新增的风格出现在下拉菜单里，生成的文章风格确实不一样 ✅

### 步骤 5：理解 localStorage 风格持久化

```typescript
const update = (partial: Partial<StyleConfig>) => {
  const newConfig = { ...config, ...partial };
  onChange(newConfig);
  localStorage.setItem("style_config", JSON.stringify(newConfig));
};
```

用户上次选了"幽默风 + 美妆赛道 + 关闭 emoji"，下次打开页面自动恢复这些设置。产品经理管这叫"记住用户偏好"。

**验证：** 选一套风格配置 → 刷新页面 → 配置还在 ✅

## ⚠️ 常见坑点

| 坑 | 现象 | 原因 | 解决 |
|----|------|------|------|
| 后端加了风格但前端不显示 | 下拉菜单还是旧的 | 前端 DEFAULT_TEMPLATES 没更新，或者没从 API 拉 | 确保前端也加了，或者依赖 `/api/styles` 返回 |
| 新增风格后生成没变化 | 文章还是原来的语气 | `STYLE_PROMPTS` 的 key 和前端传的值不匹配 | 确保三处（后端两个文件 + 前端）的名称完全一致 |
| localStorage 被清 | 刷新后配置丢了 | 浏览器清缓存或隐私模式 | 正常现象，代码里有默认值兜底 |

## 📝 总结

**本章核心要点：**
- 好的风格系统 = 少而精的选项 + 精准的 Prompt
- 新增风格要改三处：后端生成器 + 后端 API + 前端组件
- 风格数据通过 `/api/styles` 前后端同步
- localStorage 持久化用户偏好，提升体验
