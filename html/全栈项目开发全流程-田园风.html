🌱 春分 · 垦荒与播种

万物之始，先选良田，再备良种。项目之初，重在择器与立规。

🌍

选田 — 技术选型

农人种地，先看土质。全栈开发亦然：

  • 前端：React 19 + Vite + TypeScript
  • 后端：FastAPI / Express / Spring Boot
  • 数据库：PostgreSQL / SQLite / MongoDB
  • 部署：Vercel + PythonAnywhere / Railway
🌾

选种 — 项目初始化

良种饱满，秋收有望。初始化四步走：

  • 脚手架搭建（Vite / create-react-app）
  • 依赖安装与版本锁定
  • 目录结构规划（如农夫分畦）
  • Git 初始化 + .gitignore 配置
🧑‍🌾 老农口诀
"选田不精，白忙一季。选种不慎，颗粒无收。"

🌿 谷雨 · 育苗与规划

种子入土之前，先画好田垄，算好间距。软件项目，先设计架构，再动手编码。

📐

需求分析

与"地主"（客户/PM）对需求，明确边界。用 SDD 规范驱动法，写出六要素规范文档。

🏗️

架构设计

画系统架构图。前端路由树、组件树，后端 API 路由、数据模型 ER 图。如田垄分明，各司其职。

🗺️

任务拆解

将大功能拆为小任务。每个任务有明确输入输出和验收标准。如分畦而治，逐块耕耘。

🌱 育苗之道：规范驱动开发（SDD）是这阶段的最佳工具。先用 /brainstorming 理清思路，再用 /opsx:propose 生成 proposal + design + tasks 三份"农事计划书"。好的计划，让后面的耕耘事半功倍。

☀️ 夏至 · 耕耘与浇灌

日头最盛的时节，也是庄稼长得最快的时节。写代码、搭接口、连数据库，汗水浇灌，代码生长。

🗄️

数据库建模

  • 设计数据表结构
  • 建 Migration 脚本
  • 定义 Model 关联关系
  • Seed 测试数据
🔌

API 开发

  • RESTful 路由设计
  • 请求验证与错误处理
  • 鉴权中间件
  • API 文档自动生成
🎨

前端界面

  • 组件拆分与开发
  • 状态管理方案
  • 路由与页面搭建
  • API 对接与联调
☀️ 盛夏心法：用 Superpowers 的 TDD 技能——先写测试，再写代码。如农夫先挖好灌溉渠，再引水入田。测试就是你的"沟渠"，确保代码之水不泛滥、不漏失。

🐛 大暑 · 除虫与养护

庄稼长高了，虫子也来了。Bug 如蝗虫，不及时清除，满田皆毁。

🔍

系统化排虫

不要乱喷农药——用 /systematic-debugging 系统化排查：

  • 复现 Bug → 隔离范围 → 定位根因
  • 写复现测试 → 修代码 → 验证通过
  • 如捉虫先观其形、辨其类、断其巢
👀

代码审阅

请"老农"——/code-review 来田里走一圈：

  • 逻辑漏洞——如田埂缺口
  • 安全隐患——如围墙有洞
  • 代码质量——如秧苗歪斜

🌾 四时歌 · 全栈开发要诀

🌱 春不播，秋不收

需求没搞清楚就写代码，等于种子撒在荒地上。先规划，再动手。SDD 规范驱动是第一道工序。

☀️ 夏不耘，草欺苗

代码不写测试、不做 Review，Bug 就像野草一样疯长。TDD + Code Review 是日常除草必修课。

🍂 秋不验，粮生虫

功能做完了不等于能交付。验收标准逐条对照，/opsx:sync 保证文档不落后于代码。

🏠 冬不藏，明春荒

每次交付都是经验的积累。归档、写 Memory、做复盘。今天的沉淀，是明年春天的肥料。
