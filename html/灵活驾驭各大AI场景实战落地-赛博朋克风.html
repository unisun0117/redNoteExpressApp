// SCENARIO_01 · 💻 代码开发场景

MODE · ENGINEER

📐 核心打法：SDD 规范驱动

不要直接让 AI 写代码。先写规范文档（proposal + design + tasks），再让 AI 逐项实施。规范就是你和 AI 之间的"施工合同"。

  • /brainstorming → 理清需求边界
  • /opsx:propose → 生成规范文档
  • /opsx:apply → AI 按 tasks 编码
  • /code-review → 质量审查
ANTI-PATTERN

⚠️ 常见翻车姿势

  • 💬 "帮我写个登录功能" → AI 随机发挥
  • 🔄 不满意就开新对话重来 → 恶性循环
  • 📝 没有规范文档 → 一个月后看不懂
  • 🐛 不写测试 → Bug 层出不穷

正确姿势：规范→TDD→实现→Review→验证

// SCENARIO_02 · ✍️ 内容创作场景

MODE · CREATOR

🎯 核心打法：七要素公式

角色 + 任务 + 格式 + 受众 + 语气 + 示例 + 约束。七个要素拉满，AI 输出质量直接翻 10 倍。

[角色] 你是一个小红书资深博主
[任务] 写一篇露营装备种草文
[格式] 标题+痛点+3装备+总结
[受众] 25-35岁都市白领女性
[语气] 闺蜜聊天风，多用emoji
[约束] 500字以内，不要夸张
PRO TIP

💡 迭代 > 重写

第一版永远不是最终版。不要重开对话——在同一对话中持续给反馈："第二段展开一下""换个更轻松的语气""加上数据支撑"。AI 是学徒，越调越好。

// 好的内容 = 好框架 × 持续迭代

// SCENARIO_03 · 📊 数据分析场景

MODE · ANALYST

🧠 核心打法：思维链推理

数据分析最怕 AI 直接给结论。让它"先分析，再判断"——强制展示推理过程：

  • "请一步步分析这份数据"
  • "先描述你观察到的趋势"
  • "再给出可能的解释"
  • "最后给出建议"
  • "标注你不太确定的结论"
DATA PIPELINE

📡 数据获取方式

  • 📎 粘贴 CSV/JSON 原始数据
  • 🔗 Agent-Reach 抓取网页数据
  • 🐍 Python 脚本预处理后喂给 AI
  • 📊 AI 直接生成可视化代码

// 数据 + AI = 超级分析师

// SCENARIO_04 · ⚡ 自动化编排场景

MODE · ORCHESTRATOR

🔗 核心打法：链式编排

单个 AI 任务是工具，多个 AI 任务串联是生产力。Harness 的 Workflow 让你像编排微服务一样编排 AI 任务：

  • Agent → 独立执行子任务
  • Pipeline → 流水线串联处理
  • Parallel → 并行加速执行
  • Cron → 定时自动触发
REAL CASE

📋 实战：自动化日报生成

$ cron.daily_report()
09:00 → Agent_1 抓取各平台数据
09:05 → Agent_2 分析数据趋势
09:10 → Agent_3 生成日报文本
09:12 → Agent_4 推送到企业微信
[OK] 日报已自动发送
