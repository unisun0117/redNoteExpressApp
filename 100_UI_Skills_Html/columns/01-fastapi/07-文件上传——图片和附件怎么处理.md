@[TOC](目录)

## 摘要
> 本文教你用 FastAPI 处理文件上传，覆盖单文件、多文件、图片验证、文件大小限制等实战场景。适合需要让用户上传头像、图片、附件的开发者，学完你就能实现完整的文件上传接口。

## 引言

几乎所有应用都需要上传文件——用户的头像、文章里的图片、Excel 报表、PDF 合同。前端同学对 `<input type="file">` 和 `FormData` 应该不陌生，但文件选择后怎么发到后端、后端怎么存、存到哪里，这里面有不少细节。

上传文件这件事，前后端各司其职：前端负责把文件打包成 `multipart/form-data` 发给后端，后端负责接收、校验、存储、返回访问 URL。任何一个环节出问题，都会导致上传失败。

这篇文章从最基础的单文件上传讲起，到多文件同时上传，再到生产环境必备的校验和存储策略。跟着写一遍，你就再也不会被文件上传卡住了。

## 基础知识储备

- **multipart/form-data**：文件上传专用的 Content-Type。不同于 JSON（`application/json`），它可以把文件二进制和普通字段一起打包发送。
- **前端 FormData 对象**：`formData.append('file', fileInput.files[0])`，然后把 FormData 作为 fetch 的 body 发出去。
- **Python 的文件操作**：`open()`、`shutil.copy()`、`os.makedirs()`，用来把接收到的文件写到磁盘上。
- **文件路径安全**：用户上传的文件名可能包含 `../` 这种路径穿越攻击，必须做安全检查。

## 正文

### 单文件上传：最基础的用法

```python
from fastapi import FastAPI, UploadFile, File

app = FastAPI()

@app.post("/upload/")
async def upload_file(file: UploadFile = File(...)):
    # 读取文件内容
    contents = await file.read()
    
    # 保存到本地
    with open(f"uploads/{file.filename}", "wb") as f:
        f.write(contents)
    
    return {
        "filename": file.filename,
        "size": len(contents),
        "content_type": file.content_type
    }
```

前端调用代码：

```javascript
const input = document.querySelector('input[type="file"]');
const file = input.files[0];

const formData = new FormData();
formData.append('file', file);

const res = await fetch('/upload/', {
    method: 'POST',
    body: formData  // 不要设 Content-Type，浏览器会自动加 multipart/form-data
});

const result = await res.json();
console.log(result.filename, result.size);
```

几个关键点：
- `UploadFile` 是 FastAPI 提供的专门处理文件的类型，它会自动处理 multipart 解析、缓存和元数据。
- `await file.read()` 读取全部内容到内存。小文件（<1MB）没问题，大文件后面会说怎么处理。
- 前端用 `FormData` 的 `append` 方法添加文件，key（`'file'`）必须和后端参数名一致。

### 多文件上传

```python
from typing import List

@app.post("/upload/multiple/")
async def upload_files(files: List[UploadFile] = File(...)):
    results = []
    for file in files:
        contents = await file.read()
        file_path = f"uploads/{file.filename}"
        with open(file_path, "wb") as f:
            f.write(contents)
        results.append({"filename": file.filename, "size": len(contents)})
    
    return {"files": results, "count": len(results)}
```

前端传法：

```javascript
const input = document.querySelector('input[type="file"][multiple]');
const formData = new FormData();

// 注意：key 都是 'files'，和后端参数名一致
for (const file of input.files) {
    formData.append('files', file);
}

await fetch('/upload/multiple/', { method: 'POST', body: formData });
```

### 生产环境的三个必修操作

**1. 文件名校验**

用户上传的文件名可能包含路径穿越攻击（`../../../etc/passwd`），必须清洗：

```python
import uuid
import os

def safe_filename(original_name: str) -> str:
    # 只保留扩展名
    ext = os.path.splitext(original_name)[1]
    # 生成随机文件名
    return f"{uuid.uuid4().hex}{ext}"

# 使用时
safe_name = safe_filename(file.filename)
file_path = f"uploads/{safe_name}"
```

随机文件名有两个好处：防止路径穿越攻击，防止文件名冲突（两个用户都传了 `头像.png`）。

**2. 限制文件大小**

```python
MAX_FILE_SIZE = 5 * 1024 * 1024  # 5MB

@app.post("/upload/")
async def upload_file(file: UploadFile = File(...)):
    contents = await file.read()
    
    if len(contents) > MAX_FILE_SIZE:
        raise HTTPException(status_code=413, detail="文件大小不能超过 5MB")
    
    # 继续处理...
```

注意：`File(...)` 本身不支持 max_size 参数（Pydantic 的 Field 约束对 UploadFile 无效），需要手动校验。

**3. 图片类型校验**

```python
ALLOWED_TYPES = {"image/jpeg", "image/png", "image/webp", "image/gif"}

@app.post("/upload/avatar")
async def upload_avatar(file: UploadFile = File(...)):
    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(status_code=400, detail="只支持 JPG、PNG、WebP、GIF 格式")
    
    # 注意：content_type 来自客户端，可能被伪造
    # 更严谨的做法是用 python-magic 检测真实文件类型
    ...
```

### 同时接收文件和 JSON 数据

```python
@app.post("/posts/with-image")
async def create_post(
    title: str = Form(...),       # 普通表单字段
    content: str = Form(...),     # 普通表单字段
    image: UploadFile = File(...) # 文件字段
):
    contents = await image.read()
    # 保存文件和标题内容...
    return {"title": title, "image": image.filename}
```

`File` 和 `Form` 混用时，前端用 `FormData` 一把梭：

```javascript
const formData = new FormData();
formData.append('title', '我的文章标题');
formData.append('content', '文章正文...');
formData.append('image', file);

await fetch('/posts/with-image', { method: 'POST', body: formData });
```

## 总结

1. FastAPI 用 `UploadFile` 接收文件，用 `File()` 接收普通表单字段，用法清晰。
2. 单文件用 `UploadFile = File(...)`，多文件用 `List[UploadFile] = File(...)`。
3. 生产环境必须做三件事：文件名清洗（防止路径穿越）、文件大小限制、文件类型校验。
4. 文件存储有几种选择：本地磁盘（简单但不可扩展）、云存储如 S3/OSS（推荐）、数据库存二进制（不推荐，性能差）。
5. 前端用 `FormData` 打包文件和普通字段，`method: 'POST'`，不要手动设 `Content-Type`。

## 注意事项

1. 大文件（>10MB）不要用 `await file.read()` 一次性读到内存，应该用 `file.file` 对象分块读取。或者用流式上传。
2. `UploadFile` 自带 `filename`、`content_type`、`size` 属性，`content_type` 来自客户端上传时的声明，不可完全信任。
3. 上传的文件不要直接放在项目目录下的 static 文件夹里——万一部署更新覆盖了怎么办？用独立的存储目录或云存储。

## 坑点

| 坑 | 现象 | 原因 | 解决方案 |
|----|------|------|----------|
| 前端 JSON 传文件 | 后端收不到文件，字段都是空 | 文件必须用 `multipart/form-data`，不能用 `application/json` | 前端用 FormData，不要用 JSON.stringify |
| 文件名中文乱码 | 保存的文件名变成乱码 | HTTP 头编码和文件系统编码不一致 | 用 UUID 重命名，前端传原始文件名存在数据库里 |
| 大文件撑爆内存 | 上传 100MB 文件后服务直接 OOM | `await file.read()` 把整个文件加载到内存 | 用 `shutil.copyfileobj(file.file, f)` 分块写入 |
| 两个用户同时传同名文件 | 后上传的覆盖了先上传的 | 用了原始文件名 | 每个文件用 UUID 重命名 |
| 目录不存在 | `FileNotFoundError` | 代码里写了 `uploads/` 但没创建这个文件夹 | 启动时检查并创建存储目录 `os.makedirs("uploads", exist_ok=True)` |
