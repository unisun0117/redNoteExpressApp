## 引言

前端做数据可视化，chart里的每一个数字，背后都对应着数据库的一次聚合查询。你要展示"本月销售额"——SUM；展示"日均访问量"——AVG；展示"各品类销量排行"——GROUP BY + COUNT。

```javascript
// 前端reduce做聚合 —— 又慢又笨
const orders = [...]; // 10000条订单
const salesByCategory = orders.reduce((acc, order) => {
  acc[order.category] = (acc[order.category] || 0) + order.amount;
  return acc;
}, {});
```

这样写当然能跑——前提是10000条数据全在前端内存里。如果数据量是100万条呢？你不可能全拉到浏览器里reduce。聚合查询的价值就在这里：**让数据库在服务端完成计算，只把结果返回前端**。100万条数据算完返回一行数字，这就是聚合的力量。

本文覆盖五大聚合函数和GROUP BY的全部核心用法，让你能用SQL回答任何"统计类"问题。

## 基础知识储备

## 正文

### 一、五大聚合函数——SQL版的reduce

| 函数 | 作用 | JS等同写法 | 示例 |
|------|------|-----------|------|
| COUNT | 计数 | `array.length` | `COUNT(*)` 统计行数 |
| SUM | 求和 | `reduce((a,b)=>a+b, 0)` | `SUM(price)` 总销售额 |
| AVG | 平均值 | `sum / count` | `AVG(score)` 平均分 |
| MAX | 最大值 | `Math.max(...arr)` | `MAX(price)` 最高价 |
| MIN | 最小值 | `Math.min(...arr)` | `MIN(price)` 最低价 |

动手实践——建一个订单表做聚合实验：

```python
import sqlite3

conn = sqlite3.connect('agg_demo.db')
cursor = conn.cursor()

cursor.execute('''
    CREATE TABLE IF NOT EXISTS orders (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        product TEXT NOT NULL,
        category TEXT NOT NULL,
        amount REAL NOT NULL,
        quantity INTEGER NOT NULL DEFAULT 1,
        created_at TEXT DEFAULT (datetime('now'))
    )
''')

# 插入测试数据
test_data = [
    ('iPhone 15', '电子产品', 6999, 2),
    ('MacBook Pro', '电子产品', 14999, 1),
    ('Nike跑鞋', '运动户外', 899, 3),
    ('瑜伽垫', '运动户外', 129, 5),
    ('《JavaScript高级编程》', '图书', 89, 10),
    ('AirPods Pro', '电子产品', 1899, 4),
    ('羽毛球拍', '运动户外', 299, 2),
    ('《数据库系统概念》', '图书', 119, 8),
]

cursor.executemany(
    'INSERT INTO orders (product, category, amount, quantity) VALUES (?, ?, ?, ?)',
    test_data
)
conn.commit()

# 聚合查询演示
print("=== 基础聚合 ===")
cursor.execute("SELECT COUNT(*) FROM orders")
print(f"总订单数：{cursor.fetchone()[0]}")

cursor.execute("SELECT SUM(amount * quantity) FROM orders")
print(f"总销售额：¥{cursor.fetchone()[0]:.2f}")

cursor.execute("SELECT AVG(amount) FROM orders")
print(f"平均单价：¥{cursor.fetchone()[0]:.2f}")

cursor.execute("SELECT MAX(amount), MIN(amount) FROM orders")
row = cursor.fetchone()
print(f"最高价：¥{row[0]}，最低价：¥{row[1]}")

conn.close()
```

### 二、GROUP BY —— 分组统计引擎

聚合函数和GROUP BY搭档，才能发挥真正威力。GROUP BY相当于把数据按某个维度"切块"，然后对每一块分别做聚合。

```sql
-- 每个品类的销售额
SELECT category,
       SUM(amount * quantity) AS total_sales,
       COUNT(*) AS order_count
FROM orders
GROUP BY category;
```

这个查询的结果，直接就能喂给前端图表：

```javascript
// 前端echarts直接消费查询结果
const chartData = rows.map(row => ({
  name: row.category,
  value: row.total_sales
}));
// pieChart.setOption({ series: [{ data: chartData }] });
```

**分组后排序：**

```sql
SELECT category,
       SUM(amount * quantity) AS total_sales
FROM orders
GROUP BY category
ORDER BY total_sales DESC;  -- 销售额从高到低
```

**多维度分组：**

```sql
-- 每个品类每个月（如果数据跨月）的销售额
SELECT category, strftime('%Y-%m', created_at) AS month,
       SUM(amount * quantity) AS total_sales
FROM orders
GROUP BY category, month;
```

### 三、HAVING —— 分组后的"WHERE"

WHERE在分组前过滤行，HAVING在分组后过滤组。这是很多初学者搞混的地方。

```sql
-- WHERE: 分组前过滤 —— 只统计单价>100的订单
-- HAVING: 分组后过滤 —— 只保留销售额>1000的品类
SELECT category,
       SUM(amount * quantity) AS total_sales
FROM orders
WHERE amount > 100             -- 先过滤行
GROUP BY category
HAVING total_sales > 1000;     -- 再过滤组
```

**WHERE和HAVING的执行顺序：**
1. FROM → 确定数据来源
2. WHERE → 过滤行
3. GROUP BY → 分组
4. HAVING → 过滤组
5. SELECT → 选择列
6. ORDER BY → 排序

记住这个顺序，写复杂查询就不会搞混。

**Python完整示例——展示各品类排行榜：**

```python
cursor.execute('''
    SELECT category,
           COUNT(*) AS cnt,
           SUM(amount * quantity) AS sales,
           AVG(amount) AS avg_price
    FROM orders
    GROUP BY category
    HAVING sales > 500
    ORDER BY sales DESC
''')

print("\n=== 品类销售排行（销售额>500） ===")
print(f"{'品类':<10} {'订单数':<8} {'销售额':<12} {'均价':<10}")
print("-" * 45)
for row in cursor.fetchall():
    print(f"{row[0]:<10} {row[1]:<8} ¥{row[2]:<11.2f} ¥{row[3]:<9.2f}")
```

## 总结

1. 五大聚合函数（COUNT、SUM、AVG、MAX、MIN）是SQL的"reduce"——把原始行变成统计值。
2. GROUP BY按维度切分数据，每个维度独立聚合——结果直接对接前端图表数据结构。
3. WHERE过滤行（分组前），HAVING过滤组（分组后）——执行顺序是关键。
4. 聚合查询在服务端完成计算，前端只收结果——这就是数据驱动的正确姿势。
5. 聚合和JOIN可以组合——比如"每个用户的订单总额"，JOIN用户表和订单表，再GROUP BY用户ID聚合。

## 注意事项

1. SELECT中的非聚合列必须出现在GROUP BY中——这是SQL的硬性规定。`SELECT category, COUNT(*) FROM orders` 没问题，但 `SELECT category, product, COUNT(*) FROM orders GROUP BY category` 会报错，因为product不在GROUP BY里。
2. COUNT(*)和COUNT(列名)不一样——前者统计所有行（包括NULL），后者只统计该列非NULL的行。通常用COUNT(*)就好。
3. 聚合查询不能嵌套聚合函数——`MAX(SUM(amount))` 是不行的，除非用子查询或WITH语句包裹。

## 坑点
| 坑 | 现象 | 原因 | 解决方案 |
|----|------|------|----------|
| SELECT了没在GROUP BY里的列 | SQLite不报错但返回随机值，MySQL报错 | SQLite宽容但结果不可预测 | 严格遵循"SELECT的非聚合列必须都在GROUP BY里" |
| 把HAVING当WHERE用 | 写了`GROUP BY ... HAVING amount > 100`，结果和预期不符 | HAVING是分组后过滤，amount > 100应该在分组前用WHERE过滤 | 分组前过滤用WHERE，分组后过滤用HAVING |
| COUNT列名却碰到NULL | COUNT(user_id)比预期少 | NULL不参与COUNT(列名)的计数 | 用COUNT(*)统计所有行；或用COUNT(列名)配合列上的NOT NULL约束 |
| GROUP BY后的NULL被当作一组 | 所有NULL行被汇总成一行 | SQL中NULL被视为"相等"用于分组 | 在分组前用WHERE IS NOT NULL过滤掉，或者在SELECT中用COALESCE替换 |
