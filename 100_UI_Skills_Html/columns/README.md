# 后端从0到1 · 学习路线图

> 4 个专栏，40 篇文章，40 个 Lab 习题。按顺序学，不要跳。

---

## 🗺️ 学习顺序（必须按这个走）

```
第1步          第2步           第3步            第4步
FastAPI     →  REST API    →   数据库      →   用户认证
从0到1         设计从0到1       设计从0到1       从0到1
(10篇)         (10篇)          (10篇)          (10篇)

先把后端       学会设计         数据真正          加上登录
跑起来         接口规范         存下来            和权限
```

## 为什么这个顺序？

| 如果先学... | 会碰到... |
|------------|----------|
| 先学数据库再学 FastAPI | 学了 SQL 不知道怎么写成接口，枯燥劝退 |
| 先学认证再学 FastAPI | JWT 是什么都不知道，代码里全是未定义的概念 |
| 先学 REST API 再学 FastAPI | 没有后端基础，设计规范是空中楼阁 |

**正确顺序：** 先跑起来一个能用的后端（FastAPI）→ 知道怎么设计接口（REST API）→ 数据怎么存（数据库）→ 怎么保护接口（认证）。每一步都建立在上一步的基础上。

---

## 📅 建议学习节奏

```
每周 2-3 篇 + 完成 Lab → 4 个月学完全部 4 个专栏

月1: FastAPI 10篇        → 能写 CRUD 接口
月2: REST API 10篇       → 会设计规范接口
月3: 数据库 10篇          → 懂 SQL + 表设计
月4: 用户认证 10篇        → 能做登录注册权限系统
```
