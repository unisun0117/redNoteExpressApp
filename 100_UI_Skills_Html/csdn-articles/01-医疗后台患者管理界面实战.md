# 纯 HTML+Tailwind 3小时搭完一套医疗后台，Bento Grid 布局全拆解

@[TOC](目录)

## 前言

> 不需要 React、不需要 Webpack、不需要 npm install——一个 HTML 文件，7 个页面，浏览器直接打开就能跑。

最近给一个医疗健康项目做后台原型，需求是患者管理 Dashboard：核心指标卡、失约率热力图、实时就诊队列、患者档案表、用药依从性进度环。要求专业沉稳、蓝色系。

传统做法是装一堆依赖、配路由、写组件——半天过去了还没看到页面。这次我换了个思路：**一个 HTML 文件，Tailwind CDN + 原生 JS，写完即看**。效果意外地好，顺手拆解一下思路。

## 为什么选纯 HTML + Tailwind CDN

在国内开发，选技术栈有一个硬约束——**CDN 能不能稳定加载**。我踩过的坑：

```javascript
// ❌ React UMD 从 esm.sh 加载 —— 国内大概率白屏
<script src="https://esm.sh/react@18/umd/react.production.min.js"></script>

// ✅ 改用 jsdelivr（国内有 CDN 节点）
<script src="https://cdn.jsdelivr.net/npm/react@18/umd/react.production.min.js"></script>

// ✅ 最优解：不依赖框架，纯原生 JS
```

最终方案：Tailwind CSS 从 `cdn.tailwindcss.com` 加载（Cloudflare，国内通常可通），JavaScript 全部手写原生。即使 Tailwind CDN 挂了，CSS 变量兜底，页面只是没样式而不会白屏。

> 💡 **阿珊小贴士**：国内做 Demo/原型/个人项目，优先选 **零依赖单文件** 方案。框架是好东西，但等你配完环境、解决了版本冲突、处理了 CORS，灵感早跑了。

## 核心设计：CSS 变量 + 布局模式库

### 设计令牌一把梭

```css
:root {
  --bg:       #f1f5f9;   /* 页面底色 */
  --card:     #ffffff;   /* 卡片底色 */
  --border:   #e8ecf1;   /* 边框色 */
  --primary:  #3366ff;   /* 医疗蓝主色 */
  --success:  #10b981;   /* 正向/达标 */
  --warn:     #f59e0b;   /* 警告/关注 */
  --danger:   #f43f5e;   /* 危险/异常 */
  --radius:   14px;       /* 全局圆角 */
  --shadow:   0 1px 3px rgba(0,0,0,.04);
}
```

好处：换行业配色只需改 `--primary`，其他组件自动跟随。医疗用蓝、金融用金、旅游用珊瑚橙——同一个 HTML 结构改 6 个变量，气质完全不同。

### 布局不是只有 Grid 三等分

很多后台 Demo 所有页面都是 3 列等宽网格排列——浏览 30 秒就审美疲劳。我整理了一个布局模式库，每个页面用不同的：

| 模式 | 适用场景 | 视觉特征 |
|------|----------|----------|
| Hero Banner + 非对称 | Dashboard 首页 | 顶部渐变横幅吸引视线 |
| Bento Grid (5+7) | 数据密集页 | 卡片大小随内容变化 |
| 左右分栏 + 时间轴 | 日程/排班 | 主操作区 + 信息侧栏 |
| 三栏（侧-中-侧） | 分析/洞察 | 数据面板包围图表 |
| 卡片网格 | 产品/权益展示 | 同等重要级内容并排 |

> 💡 **阿珊小贴士**：每个页面换个布局 = 零成本提升专业感。老板说不出来为什么好，但就是觉得"这个看起来不一样"。

## 组件拆解：5 个高频后台组件

### 1. 指标卡 —— 数字 + 迷你折线 + 趋势徽章

```html
<div class="metric-card">
  <div class="metric-icon" style="background:#eef4ff;color:var(--primary);">👥</div>
  <div>
    <div class="metric-num">2,486</div>
    <div class="metric-label">总患者数</div>
    <span class="metric-badge up">↑ 3.2%</span>
  </div>
  <!-- 右侧迷你 SVG 折线图 -->
</div>
```

核心 CSS：
```css
.metric-num { font-size: 26px; font-weight: 800; letter-spacing: -1px; }
.metric-badge { padding: 2px 8px; border-radius: 20px; font-size: 11px; }
.metric-badge.up { background: #ecfdf5; color: #059669; }
```

### 2. 热力图 —— 纯 CSS Grid + 颜色梯度

```javascript
function getHeatColor(rate) {
  if (rate <= 0.04) return { bg: '#dbeafe', text: '#1e40af' };   // 低 → 浅蓝
  if (rate <= 0.08) return { bg: '#93c5fd', text: '#1e3a5f' };   // 中低
  if (rate <= 0.12) return { bg: '#fde68a', text: '#92400e' };   // 中 → 黄
  if (rate <= 0.16) return { bg: '#fbbf24', text: '#78350f' };   // 中高
  return { bg: '#f87171', text: '#7f1d1d' };                      // 高 → 红
}
```

7天 × 4时段 = 28 个色块，悬停放大 1.18 倍 + 阴影增强，不用任何图表库。

### 3. SVG 环形进度 —— 不用 Chart.js

```javascript
// 核心公式
const radius = (size - strokeWidth * 2) / 2;
const circumference = 2 * Math.PI * radius;
const offset = circumference - (percentage / 100) * circumference;

// SVG circle 的 stroke-dasharray = 总周长
// stroke-dashoffset = 剩余部分 = 总周长 - 已完成
// transform: rotate(-90deg) → 从 12 点钟方向开始
```

```html
<circle cx="45" cy="45" r="36" fill="none" 
  stroke="#e2e8f0" stroke-width="8"/>
<circle cx="45" cy="45" r="36" fill="none" 
  stroke="#10b981" stroke-width="8" stroke-linecap="round"
  stroke-dasharray="226" stroke-dashoffset="50"
  transform="rotate(-90 45 45)"/>
```

颜色逻辑：≥80% 绿色、60-79% 黄色、<60% 红色。

### 4. 柱状图 —— 纯 CSS + 原生 JS 渲染

```javascript
function barChart(container, values, labels, maxHeight) {
  const max = Math.max(...values);
  container.innerHTML = values.map((v, i) => `
    <div style="flex:1;display:flex;flex-direction:column;align-items:center;gap:4px;">
      <span style="font-size:10px;">${v}</span>
      <div style="width:100%;
        height:${v/max*maxHeight}px;
        background:${i===values.length-1?'var(--primary)':'#93c5fd'};
        border-radius:5px 5px 0 0;">
      </div>
      <span style="font-size:10px;color:var(--muted);">${labels[i]}</span>
    </div>
  `).join('');
}
```

最新一月用主色高亮，历史月份用浅色。

### 5. 实时队列 —— CSS 脉冲动画

```css
@keyframes pulse-amber {
  0%, 100% { box-shadow: 0 0 0 0 rgba(245,158,11,.4); }
  50%     { box-shadow: 0 0 0 7px rgba(245,158,11,0); }
}
```

等候中用琥珀色脉冲，就诊中用蓝色脉冲，已完成静态绿色——状态一目了然。

## 多页面切换：只改 innerHTML

整个系统 7 个页面，路由实现极简：

```javascript
const renderers = { dash: pageDash, patients: pagePatients, ... };

function navTo(page) {
  // 高亮当前导航
  document.querySelectorAll('.nav-item').forEach(b => b.classList.remove('active'));
  document.querySelector(`[data-pg="${page}"]`).classList.add('active');
  
  // 渲染页面
  document.getElementById('content').innerHTML = renderers[page]();
  
  // 初始化该页的交互数据
  initPage(page);
}
```

每个页面是一个返回 HTML 字符串的纯函数，切换时直接替换 `innerHTML`。没有虚拟 DOM diff、没有路由库、没有状态管理——页面不复杂的时候，越简单越稳。

## 踩过的坑

| 坑 | 现象 | 解法 |
|----|------|------|
| Python 语法混入 JS | `for d in days` 导致白屏 | 写完检查：`for...in`/列表推导式/f-string 全部替换 |
| 函数忘记闭合 `}` | 模板字符串结尾漏了函数大括号 | 不依赖 IDE，手动数 `{` 和 `}` 配对 |
| `esm.sh` CDN 在国内不稳定 | React UMD 加载失败 | 换成 jsdelivr 或直接用原生 JS |
| `initPage()` 中访问了其他页面的 DOM | `getElementById` 返回 null | 每个页面初始化前加 `if(!el) return` |
| Google Fonts 加载超时 | 字体不渲染 | CSS font-family 中放系统字体做 fallback |

## 总结

这套方案的真正价值不是"不用框架"，而是**把搭建后台 Demo 的门槛降到最低**：

1. **零依赖单文件** — 浏览器直接打开，不需要 Node.js 环境
2. **CSS 变量做主题** — 换行业配色只改变量，不用改组件
3. **布局模式库** — 7 个页面 7 种布局，不重复才专业
4. **原生 JS 组件** — 热力图、环形图、柱状图不需要任何图表库
5. **CDN 白名单** — 知道哪些 CDN 国内能通，避免白屏事故

完整代码 700+ 行，包含了医疗、金融、旅游、PM、ERP 五套不同行业的后台系统，需要的自取。
